经全面检索，该仓库为「目标&达成管理平台」的**纯文档与协作宪法仓库**，不包含任何源代码、构建脚本或 CI/CD 配置。仓库根目录仅包含 `.gitignore`、`LICENSE`、`CLAUDE.md`（项目工作手册）以及 `docs/` 下的需求、设计、ADR 等文档。

从 `CLAUDE.md` 可知项目的构建约定如下：
- **后端**：基于 Spring Cloud Netflix 的公司内部框架 Hulk，使用 Maven 构建（`mvn clean package` / `mvn test` / `mvn verify`），父 BOM 为 `com.transsion.hulk:hulk-parent:4.0.0.RELEASE`。
- **前端**：基于 Vben Admin（Vue3 + Vite + TypeScript + Ant Design Vue），使用 pnpm monorepo + turbo，命令包括 `pnpm install`、`pnpm dev`、`pnpm build`、`pnpm lint`、`pnpm type:check`、`pnpm test`、`pnpm commit`。
- **部署**：通过公司网关 + Eureka 注册发现，外部系统调用经 OpenFeign。

但这些构建配置（pom.xml、package.json、Dockerfile、CI 流水线等）**不在本仓库中**，应位于各自的前端/后端工程仓库内。本仓库仅作为文档与协作规范的单一事实来源。

因此，对于「build_system」这一类别，本仓库**不适用**——它没有实际的构建系统代码，只定义了构建规范。
