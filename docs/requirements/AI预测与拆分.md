# AI 预测与拆分 — 需求规格

> 所属项目：AI 目标管理平台（孟加拉一期试点）
> 模块编号：M02
> 版本：v1.0
> 日期：2026-07-02
> 状态：基线（来源：待澄清问题清单.md 澄清结论，权威口径）

> **AI 框架与预测来源（决策 F1 + F2）**：本模块 AI 能力经 **AgentScope Java（LLM Agent 框架）** 实现，全程人工确认（HITL）。**预测基准值由大数据团队实时调用返回；三档（保守/推荐/挑战）与推荐区间由本系统按配置规则在基准上生成**（F2；本系统不自建 Prophet/SARIMA 等模型、不需数据科学家建模）；"业务语言依据、复盘、策略"文本由 LLM Agent 生成。AI 只做建议、不自动决策。新增依赖：大数据团队**实时预测接口** + LLM Key/成本/**数据出境合规**（见待澄清 F1/F2）。具体 Agent/工具与接口边界属设计期（ADR/design）。

## 1. 模块目标与范围边界

**模块目标**：基于 DRP/DCR/激活/库存历史数据，为目标制定人提供 AI 三档总目标预测建议，并在总目标确认后按简单规则自动拆分至各层级（国家→区域→目标对象(Dealer/门店/Promoter)→机型），过强校验后向"目标协同管理"模块移交。

### 做什么

- 预测基准值：**实时调用大数据团队获取**；三档（保守/推荐/挑战）+ 推荐区间：**本系统按配置规则在基准上生成**；业务语言依据由 LLM Agent 生成（本系统不自建预测模型）
- 数据充分性检查与降级提示（不允许伪精确）
- 多级多维规则拆分（一期仅简单规则：历史达成率/资金盘/阵地数/过去3月平均销量）
- 自动平衡（四种模式，已锁定值不参与）
- 强校验（三级合计 ≥ 约束，0 容忍）
- 拆分阶段异常识别（一期 4 类有效，2 类因数据缺失降级，见第 6 节）
- 高风险（红色）异常的人工处理卡点
- 所有人工调整留痕（操作人/时间/调整前后值/原因）

### 不做什么（一期）

- 审批流 / 加签 / 转交（二期接 BPM）
- 对象移动端确认与申请调整（二期）
- 超时 Pass / 催办机制（二期）
- 市场容量因子、机型生命周期因子（二期，市场库为二期数据源）
- "超市场容量"异常识别（市场库为二期数据源，一期整体降级）
- "新品供应风险"异常识别（工厂产能数据为二期数据源，一期整体降级）
- 版本对比 UI（仅后端留痕）
- 独立 AI 对话助手入口 / 10-Agent 产品化编排能力（数据沉淀后二期实现；一期仅使用 LLM 生成业务依据等文本，不提供独立助手入口）
- 返利计算与返利偏差率指标（不涉及）
- 预测准确率指标（一期合理性指标统一为"目标达成率占比"）
- 防作弊 / 串货检测（允许跨区域，不检测）
- **总目标的确认操作**（属于"目标协同管理"模块）
- **拆分确认与发布**（属于"目标协同管理"模块）

## 2. 功能需求

### 2.1 用户故事

| # | 角色 | 能力 | 价值 |
|---|------|------|------|
| US-01 | 目标创建人 | 查看 AI 三档预测及业务语言依据 | 有据可依地拍板或调整总目标，而不是依靠经验拍脑袋 |
| US-02 | 目标创建人 | 当数据不足时收到明确的降级提示 | 知晓预测可信度，不被虚假置信度误导决策 |
| US-03 | 目标创建人 | 系统按配置规则自动拆分总目标到各层级 | 将人工拆解工时从 2-3 天压缩到 5 分钟内 |
| US-04 | 目标创建人 | 手动调整某层目标值后触发自动平衡 | 下级合计自动重新满足上级约束，无需逐条手动计算 |
| US-05 | 目标创建人 | 查看异常识别结果并处理（一期均为红色异常，强制处理后方可提交） | 在提交前发现并处置风险点，避免目标失真 |
| US-06 | 目标创建人 | 偏离 AI 区间时必须填写调整原因才能提交 | AI 建议不被随意忽视，调整有据可查 |
| US-07 | 系统管理员 | 按国家/品牌/目标类型配置拆分权重规则 | 适应各地区业务差异，无需改代码 |
| US-08 | 目标创建人 | 强校验不通过时看到具体层级名称和差额 | 快速定位哪一层合计不满足约束并修正 |

### 2.2 主流程

```
前置条件：
  - 历史数据已接入（DRP/DCR/激活/库存），外部系统延迟 ≤ 24 小时
  - 目标草稿已由"目标协同管理"模块创建（含国家/品牌/目标类型/目标周期）

[T-7] Step-1 数据拉取与充分性检查（本模块）
  → 检查近 12 个月历史数据完整度
  → 输出**预测数据充分性等级**：FULL / PARTIAL / LIMITED
  → LIMITED：写入降级标记，待预测时携带提示

[T-5] Step-2 获取预测并生成三档（本模块核心）
  → 实时调用大数据团队获取预测基准（特征工程/建模在大数据团队侧，见 F2）
  → 按配置规则在基准上生成三档/推荐区间
  → 输出：保守值 / 推荐值 / 挑战值 / 推荐区间[下限,上限] / 置信度 / 业务语言依据
  → 若数据充分性为 LIMITED：显示降级提示，置信度体现数据状态，禁止伪精确

[T-5] Step-3 人工确认总目标
  → 属于"目标协同管理"模块，本模块仅提供预测结果，等待该模块回传确认值

[T-4] Step-4 AI 规则拆分（本模块核心）
  → 接收确认后的总目标值
  → 读取拆分规则配置（历史达成率/资金盘/阵地数/过去3月平均销量的权重比例）
  → 逐层拆分：国家 → 区域 → 目标对象(Dealer/门店/Promoter) → 机型
  → 拆分初稿写入拆分结果表

Step-5 拆分阶段异常识别
  → 对每个层级对象的拆分值做异常检查（见第 6 节）
  → 红色异常：标记高亮，人工必须处理或填写原因
  → （一期无黄色异常；黄色标记与"确认知晓绕过"为二期预留，见第 6 节）

Step-6 人工调整与自动平衡
  → 人工可修改任意层级的拆分值（锁定值除外）
  → 修改后触发自动平衡（按选定平衡模式重新分配未锁定部分）
  → 平衡完成后重新执行异常识别

Step-7 强校验
  → 检查三级合计约束（见第 5 节）
  → 通过：状态流转至"待发布"，移交"目标协同管理"模块处理确认与发布
  → 不通过：禁止提交，提示具体层级名称和差额，返回 Step-6

全流程留痕：
  - AI 预测快照、拆分初稿、各次人工调整均后端留版本记录
  - 版本对比 UI 一期不做，数据结构一期完整留存供二期使用
```

### 2.3 关键分支与异常场景

| 场景 | 触发条件 | 系统行为 |
|------|---------|---------|
| 数据不足降级 | 预测数据充分性为 LIMITED（历史 < 6 个月或关键字段缺失） | 显示降级提示；置信度标注数据状态；不阻断流程，但禁止显示虚高置信度 |
| AI 预测失败 | 预测服务异常或数据完全不可用 | 显示失败原因；允许用户跳过 AI 建议直接人工输入总目标（跳过动作留痕） |
| 无拆分规则配置 | 该国家/品牌/目标类型无对应配置 | 提示"该场景无拆分规则配置，请联系管理员"；阻止自动拆分 |
| 权重来源数据不可用 | 资金盘或阵地数数据缺失 | 该权重自动归零并重新归一化；界面提示"XX 权重数据不可用，已按剩余权重重新分配" |
| 修改锁定值 | 用户尝试修改已锁定目标值 | 禁止修改；提示"该值已锁定，不参与调整" |
| 平衡无法收敛 | 锁定值之和 ≥ 上级目标 | 提示"锁定值之和已超出或等于上级目标，无法平衡未锁定部分，请检查锁定设置"；禁止提交 |
| 强校验失败 | 下级合计 < 上级 | 禁止提交；提示"[区域名称] 合计(X) < 总目标(Y)，差额 Z" |

## 3. AI 能力与降级规则

### 3.1 预测输出规格

| 输出项 | 格式 | 约束 |
|--------|------|------|
| 保守值 | 数值 | ≥ 0；≤ 推荐值 |
| 推荐值 | 数值 | ≥ 保守值；≤ 挑战值 |
| 挑战值 | 数值 | ≥ 推荐值 |
| 推荐区间 | [下限, 上限] | 下限 ≤ 推荐值 ≤ 上限 |
| 置信度 | 百分比 0-100% | 体现数据充分性；LIMITED 时不得虚高 |
| 业务语言依据 | 文本（中/英） | 说明影响预测的关键因子，不可为空 |
| 预测基准来源 | 标识 | 大数据团队（实时调用）；三档/区间由本系统规则生成 |
| 预测数据充分性 | Enum | FULL / PARTIAL / LIMITED |

所有字段均不允许为空后输出——若预测失败则整体标记 FAILED，不输出部分字段。

### 3.2 预测获取原则（F2：预测来自大数据团队）

- **本系统不自建预测模型、不做特征工程/模型训练**；预测**基准值实时调用大数据团队获取**，**三档（保守/推荐/挑战）与推荐区间由本系统按配置规则在基准上生成**。
- 特征工程、模型选择与训练均由**大数据团队**负责；本侧**不需要数据科学家建模**。
- 获取到的预测来源标识、对应周期与获取时间写入预测结果留痕，支持后续回溯。

### 3.3 预测数据充分性分级与降级规则

| 预测数据充分性等级 | 定义 | 系统行为 |
|------------------|------|---------|
| FULL（完整） | 近 12 个月历史数据无重大缺口，字段完整度 > 90% | 正常预测，输出完整三档 + 真实置信度 |
| PARTIAL（部分） | 历史数据 6-12 个月，或字段有缺口但主干完整 | 预测输出，置信度降低；界面显示"数据部分可用，置信度受影响" |
| LIMITED（受限） | 历史数据 < 6 个月，或关键字段缺失 | 显示"数据不足，预测仅供参考，建议结合业务经验人工设定"；**禁止显示虚高置信度（伪精确）** |

降级原则（来自 4.5 第 3 条）：数据不足时必须降级提示，不允许伪精确；系统不因数据不足拒绝给出预测，但必须清晰标注可信度状态。

> **F2 影响**：预测充分性/置信度随大数据团队返回的预测结果确定；上表等级作为展示映射与降级口径。**若大数据团队未返回预测**（或标注数据不足），系统按 LIMITED 降级为"建议人工设定"，不阻塞流程。具体以预测接口契约为准。

### 3.4 AI 定位约束（避免 AI 跑偏）

1. AI 预测为**建议值**，不直接替代人工决策；总目标由目标创建人拍板（来自 4.5 第 1 条）。
2. 所有预测结果**必须附带业务语言依据和置信度**，不允许裸输出数值（来自 4.5 第 2 条）。
3. 数据不足时**必须降级提示**，不允许伪精确（来自 4.5 第 3 条；见 3.3）。
4. 所有人工调整**必须留痕**（调整人/时间/调整前值/调整后值/原因），作为模型纠偏依据（来自 4.5 第 4 条）。
5. **高风险（红色）异常必须人工处理或填写原因后才能提交**，不允许系统静默绕过（来自 4.5 第 5 条）。
6. AI 预测结果快照入库，一期完成数据留存，支持二期模型回流优化（来自 4.5 第 6 条）。

## 4. 拆分与平衡规则

### 4.1 拆分层级（孟加拉一期固定）

```
国家 → 区域 → 目标对象(Dealer/门店/Promoter) → 机型
```

各层级之间的分配由拆分规则配置决定。目标对象为客户(Dealer)、门店、人员(Promoter)三类，同一层级并列，按目标创建时选定的对象维度展开。

### 4.2 一期拆分权重规则（简单规则）

| 权重来源 | 说明 | 一期数据来源 |
|---------|------|------------|
| 历史达成率 | 近 N 期（可配置）各对象目标达成率 | DCR/DRP 历史数据 |
| 资金盘 | 各 Dealer/门店的采购额或信用额度 | DRP 数据（待 architect 确认接口，见 OQ-02） |
| 阵地数 | 各 Dealer/门店管理的门店/Promoter 数量 | 人员/门店管理系统（待 architect 确认接口，见 OQ-03） |
| 过去3月平均销量 | 过去 3 个自然月各对象月均销量占比 | DCR/DRP 历史销量数据 |

权重组合规则：
- 同一国家/品牌/目标类型可配置各权重的占比，四项之和 = 100%。
- 若某权重来源数据不可用，系统自动将该权重归零并重新归一化，同时界面提示（见 AC-07）。
- **一期明确不含**：市场容量、机型生命周期因子（二期扩充）。

### 4.3 自动平衡规则

**一期触发时机**（来自 7.12，过滤一期不做项后）：

| # | 触发场景 | 说明 |
|---|---------|------|
| 1 | 人工修改总目标后 | 总目标变化，需重新拆分 |
| 2 | 人工修改区域层目标值后 | 区域内部重新平衡 |
| 3 | 人工修改机型层目标值后 | 对象下机型重新平衡 |
| 4 | 月中调整触发目标更新后 | 月中目标变更场景 |

不触发（一期不做）：对象申请调整被同意（移动端确认二期）、审批退回修改（审批二期）。

**平衡模式**（四选一，拆分前由创建人选择）：

| 模式 | 分配逻辑 |
|------|---------|
| AI 权重 | 按 AI 初次拆分的原始权重比例重新分配 |
| 历史销量 | 按近 3 月各对象历史销量占比分配 |
| 达成率 | 按近 3 期各对象平均达成率占比分配 |
| 平均分摊 | 等分到所有未锁定对象 |

**锁定规则**：
- 已人工手动设置"锁定"标记的目标值，在自动平衡时保持不变。
- 平衡计算：(上级目标 - 已锁定值之和) 按选定模式分配给未锁定对象。
- 若锁定值之和 ≥ 上级目标：禁止平衡并提示，见 AC-10。

### 4.4 拆分规则配置的归属与管理（方案 B，已确认）

拆分权重规则（历史达成率/资金盘/阵地数/过去3月平均销量占比、平衡模式、拆解层级等，即 `SplitRuleConfig`）**归本模块（AI 预测与拆分）独立的"拆分规则配置"能力**，**不并入**目标协同管理的流程配置后台。目标协同流程配置只管"是否拆解/拆解至层级/是否拆到机型"的流程编排；权重属拆分算法参数，与拆分逻辑内聚，故独立维护。

| 维度 | 一期规则 |
|------|---------|
| 配置入口 | 系统管理员在本模块"拆分规则配置"后台维护 `SplitRuleConfig`（按国家+品牌+目标类型组合） |
| 权限 | 仅系统管理员可增改；四项权重之和必须 = 100%，保存时强校验 |
| 审计 | 每次变更写操作日志：操作人/时间/配置维度（国家/品牌/类型）/变更前后值/`config_version` |
| 版本 | `config_version` 每次变更递增，历史配置留存可回溯 |
| 生效范围 | 新建目标触发拆分时读取当前生效版本；**已生成的拆分结果不追溯**，即改配置不回改历史拆分 |

#### 规则包优先级匹配（D-4 第三轮澄清新增，对齐产品 Demo 的 P0–P3 规则模板体系）

产品 Demo 展示了"全球标准包（P0）→ 国家专属（P1）→ 品牌/品类（P2）→ 全球兜底（P3）"的规则匹配层级，用于支撑"全球复制=新增国家规则包，不改代码"。一期按此思路落地**两级**：

- **P1 国家专属包**：孟加拉 TECNO/Infinix 的规则配置，按"国家+品牌+目标类型"精确匹配。
- **P3 全球兜底包**：国家/品牌维度取通配值，任一新国家/新品牌未配 P1 专属包时命中此包，保证不阻塞目标创建。
- 匹配顺序：先查 P1 精确匹配，未命中或未启用 → 回退查 P3 兜底包；两者都未命中才触发 `SPLIT_NO_RULE`（AC-08）。
- P0（临时指定）、P2（品牌/品类）为**匹配机制预留插级点**，一期不建配置管理界面，不影响本节其余规则。

## 5. 强校验规则

强校验在用户点击"提交"时执行，**0 容忍**，任一规则不通过禁止提交。

| 编号 | 规则 | 说明 | 违反后果 |
|------|------|------|---------|
| CV-1 | 区域目标合计 ≥ 总目标 | 所有区域分配值之和 ≥ 国家总目标 | 禁止提交；提示"区域合计(X) < 总目标(Y)，差额 Z" |
| CV-2 | 对象目标合计 ≥ 区域目标 | 每个区域下所有 Dealer/门店/Promoter 分配值之和 ≥ 该区域目标 | 禁止提交；提示具体区域名及差额 |
| CV-3 | 机型目标合计 ≥ 对象目标 | 每个对象下所有机型分配值之和 ≥ 该对象目标 | 禁止提交；提示具体对象名及差额 |

说明：采用 "≥" 模式，允许超额（下级合计 > 上级不报错）；严格相等模式留待二期。

校验通过后状态流转至"待发布"，由"目标协同管理"模块处理后续确认与发布。

## 6. 拆分阶段异常规则

### 6.1 异常类型一期实现情况

| 异常类型 | 一期实现 | 触发条件 | 预警级别 | 可绕过 | 处理策略 |
|---------|---------|---------|---------|-------|---------|
| 偏离 AI 区间 | 实现 | 拆分值（含人工调整后）超出 AI 推荐区间 [下限, 上限] | 红色 | 否，**填写调整原因后**可提交 | 标红高亮；禁止提交；强制要求填写调整原因；未填写则提交按钮不可用 |
| 偏离历史过大 | 实现 | 该对象拆分值与近 3 月历史均值偏差 > 50% | 红色 | 否，**必须人工确认处理** | 标红高亮；展示历史对比（近 3 月均值 / 当前值 / 偏差%）；要求人工确认后方可提交 |
| 超市场容量 | **不实现（一期降级）** | 区域目标 > 市场容量上限 | — | — | 市场库为二期数据源，一期无数据，整体降级至二期，界面不显示此异常类型 |
| 合计不平衡 | 实现（强校验兜底） | 下级合计 < 上级目标 | 自动提示 | 否，禁止提交 | 由强校验规则 CV-1/2/3 兜底；拆分页面实时显示合计值与差额（无需等到点提交） |
| 新品供应风险 | **不实现（一期降级）** | 新品分配量 > 工厂供应能力 | — | — | 工厂产能数据为二期数据源，一期无数据，整体降级至二期，界面不显示此异常类型 |
| 清尾风险 | **不实现（一期不做，见 P3）** | 尾货机型目标 > 历史清库速度 | — | — | 一期不做，界面不显示此异常类型 |

> **一期拆分阶段无黄色异常（已确认）**：一期实际生效的异常仅"偏离 AI 区间""偏离历史过大"（均红色）+ "合计不平衡"（强校验兜底），拆分页面**不存在黄色异常与"确认知晓绕过"流程**。下文 6.2/6.3 中的黄色处置、`SplitAnomaly.severity=YELLOW`、`bypass_confirmed` 均为**二期预留**，一期不启用、界面不出现。

### 6.2 异常分级处置机制

| 预警级别 | 行为 | 绕过条件 |
|---------|------|---------|
| 红色 | 标红高亮；展示原因说明；强制拦截提交 | 偏离 AI 区间：填写调整原因后可提交；偏离历史过大：人工确认处理后可提交；合计不平衡：修正数据使 CV-1/2/3 通过 |
| 黄色（二期预留，一期无此级别） | 标黄提示；展示风险说明；不强制拦截 | 用户点击"确认知晓"后可提交，系统记录确认日志 |

### 6.3 调整留痕规则

- 人工修改任何层级拆分值时，若该值触发红色异常，**必须填写调整原因**才能保存。
- （二期预留）黄色异常确认绕过时，记录"用户知晓确认"事件（用户/时间/异常类型）；一期无黄色异常，本项不触发。
- 所有人工调整（含无异常时的主动修改）记录：操作人 / 时间 / 调整前值 / 调整后值 / 原因（非异常场景原因可选填）/ 关联异常类型。

### 7.2 状态流转

**AI 预测状态**：
```
PENDING（待预测）→ RUNNING（预测中）→ COMPLETED（完整预测完成）
                                    → DEGRADED（降级完成，数据不足）
                 → FAILED（预测失败，记录原因）
```

**拆分状态**：
```
PENDING（待拆分）
→ SPLITTING（拆分计算中）
→ DRAFT（初稿，含异常标记，等待人工处理）
→ ADJUSTING（人工调整/平衡中）
→ VALIDATION_PASS（强校验通过，移交待发布）
→ VALIDATION_FAIL（强校验失败，返回 ADJUSTING）
```

## 10. 验收标准

### AI 预测

**AC-01** 当目标制定人打开目标草稿时，系统展示保守值/推荐值/挑战值/推荐区间/置信度/业务语言依据，六项均不为空且格式正确（见 3.1 约束），则验收通过。

**AC-02** 当预测数据充分性为 LIMITED 时，界面显示"数据不足，预测仅供参考"提示，且展示的置信度不超过 LIMITED 等级对应的合理上限，不允许显示虚高置信度。

**AC-03** 当预测数据充分性为 PARTIAL 时，界面显示"数据部分可用，置信度受影响"提示，预测三档正常输出。

**AC-04** 当 AI 预测服务异常或数据完全不可用（FAILED 状态）时，系统显示失败原因文本，并允许用户跳过 AI 建议直接人工输入总目标；该跳过动作写入操作日志。

**AC-05** 保守值 ≤ 推荐值 ≤ 挑战值，且推荐区间下限 ≤ 推荐值 ≤ 推荐区间上限；任一关系违反则系统校验不通过，标记预测为 FAILED 并降级为人工设定。

### 规则拆分

**AC-06** 当总目标确认后，系统按已配置权重规则自动生成拆分初稿，计算完成时间 ≤ 30 秒；拆分结果覆盖国家→区域→目标对象(Dealer/门店/Promoter)→机型全部层级。

**AC-07** 当某项权重来源数据不可用时，系统将该权重自动归零并重新归一化，界面显示"[XX 权重]数据不可用，已按剩余权重重新分配"，拆分仍可继续运行。

**AC-08** 当目标计划对应的国家/品牌/目标类型既无 P1 国家专属规则配置、也无 P3 全球兜底规则配置时，系统阻止自动拆分并提示"该场景无拆分规则配置，请联系管理员"。

**AC-08c**（D-4 新增）当目标计划的国家/品牌无 P1 专属配置但存在已启用的 P3 全球兜底配置时，系统按 P3 兜底配置正常执行拆分，不视为"无规则配置"、不触发 AC-08 阻止。

**AC-08b** 系统管理员可在本模块"拆分规则配置"后台按"国家+品牌+目标类型"维护四项权重占比与平衡模式（方案 B）；仅系统管理员可增改，四项权重之和 ≠ 100% 时保存被拦截；每次变更写操作日志（操作人/时间/配置维度/变更前后值/config_version），且变更仅对变更后新建目标的拆分生效，不追溯已生成的拆分结果。

### 自动平衡

**AC-09** 当人工修改某层目标值后系统触发平衡，未锁定的同级对象按选定平衡模式重新分配，平衡完成后该层下级合计满足强校验对应约束。

**AC-10** 当已锁定目标值之和 ≥ 上级目标时，系统禁止提交并提示"锁定值之和已超出或等于上级目标，无法平衡，请检查锁定设置"。

**AC-11** 执行自动平衡的全过程中，所有设置了锁定标记的目标值保持不变。

### 强校验

**AC-12** 当用户点击提交时，系统执行 CV-1（区域合计 ≥ 总目标）、CV-2（对象合计 ≥ 区域目标）、CV-3（机型合计 ≥ 对象目标）三项校验；全部满足则进入待发布状态；任一不满足则禁止提交并提示具体层级名称和差额。

**AC-13** 强校验允许下级合计超出上级（超额场景不报错，不阻止提交）。

**AC-14** 强校验的误判率为 0：所有应被拦截的提交均被拦截，所有合法提交均不被误判拦截（边界值测试验证）。

### 拆分阶段异常

**AC-15** 当拆分值（含人工调整后）超出 AI 推荐区间时，系统将该行标红，提交按钮置为不可用，且提示"超出推荐区间，请填写调整原因"；调整原因文本框填写后方可激活提交。

**AC-16** 当某对象拆分值与近 3 月历史均值偏差 > 50% 时，系统将该行标红，展示近 3 月均值、当前值、偏差百分比，并要求人工点击"确认处理"后方可提交。

**AC-17** 系统在任何情况下不触发、不展示"清尾风险"异常（一期不做，见 P3）。

**AC-18** 系统在任何情况下不触发、不展示、不拦截"超市场容量"和"新品供应风险"两类异常（一期降级，不存在该逻辑）。

**AC-19** 所有人工调整操作（含异常处理和正常修改）的操作人/时间/调整前值/调整后值/原因完整写入数据库，可通过后台日志查询追溯。

### AI 定位与留痕

**AC-20** AI 预测结果展示区明确标注"仅供参考，最终目标由您决定"（或等效表述），不存在"AI 已自动确认目标"等替代决策的表述。

**AC-21** 当存在任意未处理的红色异常时，提交操作不可执行（按钮禁用或点击后弹窗拦截并列出未处理的红色异常列表）；直至所有红色异常处理完毕或填写调整原因后，提交方可执行。

**AC-22** AI 预测快照（含预测基准来源、获取时间、三档生成规则版本、输出值）在预测完成后写入版本记录表，后续可查询。

| 风险 | 概率 | 影响 | 应对 |

## 12. 明确不做项（一期边界）

1. **审批流 / 加签 / 转交**：二期接 BPM 系统。
2. **对象移动端确认与调整申请**：二期，由 DCR APP RN 团队实现。
3. **超时 Pass / 催办机制**：二期。
4. **市场容量因子 / 机型生命周期拆分权重**：二期（市场库为二期数据源）。
5. **超市场容量异常识别**：二期（见 AC-18）。
6. **新品供应风险异常识别**：二期（见 AC-18）。
7. **版本对比 UI**：仅后端留痕，二期实现前端差异对比高亮。
8. **独立 AI 对话助手入口 / 10-Agent 产品化编排能力**：二期（需数据沉淀）；一期仅使用 LLM 生成业务依据等文本，不提供独立助手入口。
9. **返利计算 / 返利偏差率指标**：不涉及。
10. **预测准确率指标（AI 推荐值 vs 实际完成值偏差）**：一期合理性指标统一为"目标达成率占比"。
11. **防作弊 / 串货检测**：允许跨区域，一期不检测。
12. **总目标确认交互**：属于"目标协同管理"模块。
13. **拆分发布确认**：属于"目标协同管理"模块。
