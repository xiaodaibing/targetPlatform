## 第四轮扫描（2026-07-23，MVP 重定基线后）

> 触发条件：gate 2b —— 口径变更后必扫。本轮口径变更幅度为历轮最大（E-1~E-12，AI 全线移除、强校验作废、层级收窄、审批引入、双指标启用），旧表述残留风险极高。
> 扫描范围：`docs/product-vision.md`、`docs/requirements/*`（4 模块 + 基线清单）、`docs/design/*`（4 模块 + 3 契约基线）、`docs/adr/*`、`CLAUDE.md`。
> 方法：按 `templates/consistency-scan-template.md` 逐词 grep，命中行逐条核对是否为「作废声明」还是「残留」。

### 一、本轮决策基线（单一事实来源）

| 类别 | 基线口径 |
|------|---------|
| AI 能力 | **一期完全不引入**：无 LLM / Agent / AgentScope / 三档 / 置信度 / 推荐区间 / 业务语言依据 / 市场情报 / 复盘建议 |
| 校验 | **无阻断式强校验**；仅提示性校验（合计 ≥ 总目标×90%，不阻断） |
| 层级 | **国家 + 人员两级**；无区域 / Dealer / 门店层目标（门店仅作 BP 归属属性） |
| 平衡 | **无自动平衡、无锁定机制** |
| 审批 | 总目标**不审批**；人员目标**必须审批，接公司 BPM** |
| 复盘 | **一期不做**复盘报告与结果回流 |
| 范围 | 孟加拉 / **仅 Infinix** / **仅 SO** / 月度 / 目标对象仅人员 |
| 双指标 | `SALES_VOLUME` + `HSP_VOLUME`；HSP 两模式 `BY_MODEL` / `TOTAL` |
| 预警 | 仅 2 类：达成率 < 时间进度×30%、连续 N 天 0 销量 |
| 预测来源 | 大数据只供**历史指标**；总目标由本系统**公式化测算** |
| 命名 | 平台「目标&达成管理平台」；M02 `calc`（原 `forecast`）；M04「执行追踪与预警」（原「与复盘」） |

### 二、关键词扫描结果

| 关键词 | 命中文件数 | 判定 |
|--------|-----------|------|
| `强校验` `CV-1` | 8 / 3 | ✅ 全部位于作废声明与「不做项」表格，无残留 |
| `自动平衡` `锁定` | 5 / 6 | ✅ 全部为作废声明；`锁定` 另有一处为「无历史状态锁定」（tracking 快照重算语义），非同义词，放行 |
| `三档` `保守` `挑战值` `置信度` `推荐区间` | 9 / 5 / 1 / 7 / 3 | ✅ 全部为作废声明或 ADR-0002 修订背景；`挑战值` 仅 1 处（需求 §1 作废清单） |
| `AgentScope` `LLM` | 7 / 11 | ✅ 全部位于废弃 ADR、作废声明、ArchUnit 禁止清单 |
| `复盘` `回流` | 11 / 9 | ✅ 全部为「一期不做」声明或 E-4 决策引用 |
| `TECNO` | 4 | ✅ 均为「一期仅 Infinix」的对照表述或枚举保留说明 |
| `Dealer` | 10 | ✅ 均为「不展示 / 不做 / 无此层」的排除表述 |
| `区域` | 10 | ✅ 均为排除表述或肯尼亚二期链路举例 |
| `forecast` | 6 | ✅ 均为「原包名，已更名 calc」的映射说明 |
| `AI 目标管理平台` | 4 | ✅ 均为「原名，已更名」说明 |

**结论：无口径残留。** 所有命中均为有意保留的作废声明（保留是必要的——不写明「什么已作废」，实现期会有人从旧文档或旧代码里捡回来）。
