## 1. 执行追踪中的“黄色预警”是否需要触发通知

### 问题描述

`执行追踪与复盘.md` 当前同时存在两种口径：

- 口径 A：一期正式执行异常只有 4 类，其中黄色异常只有 `A4 高 DOS`
- 口径 B：所有“达成率 70%-85% / 偏差率 15%-30%”的黄色状态都需要触发通知并由 RSM 跟进

两种口径会直接影响：

- 通知量级
- `NotificationEvent` 触发条件
- 幂等规则
- RSM 接收范围
- 看板“状态色”与“异常事件”的边界

### 涉及文档

- `docs/requirements/执行追踪与复盘.md`

### 建议

明确区分“状态色”和“异常事件”：

- 若黄色仅为看板状态，则不自动发通知
- 若黄色也要发通知，则需将其补充为正式异常类型，而不是仅依附于状态色规则

**✅ 结论（已闭环）**：采口径 A。黄色为看板状态色、不发通知；一期唯一黄色异常为 A4 高 DOS，仅它触发 RSM 通知。已改 5.2 表（按异常事件分级 + 状态色/异常事件区分说明）与 AC-14。

## 2. 拆分阶段黄色异常表述残留，与一期范围不一致

### 问题描述

`AI预测与拆分.md` 已明确一期不做以下异常：

- 超市场容量
- 新品供应风险
- 清尾风险

这意味着一期拆分阶段实际只保留红色异常与强校验兜底项。但文档中仍保留：

- “黄色异常可确认知晓后绕过”
- `bypass_confirmed` 字段
- `目标协同管理.md` 中“红色或黄色异常项逐条处理”的表述

当前会造成实现方误判：以为一期拆分页面仍需支持黄色异常处理流程。

### 涉及文档

- `docs/requirements/AI预测与拆分.md`
- `docs/requirements/目标协同管理.md`

### 建议

若一期拆分阶段确无黄色异常，应删去或显式标记为“二期预留未启用”的相关表述、字段和验收项。

**✅ 结论（已闭环）**：一期拆分无黄色异常。字段（severity 枚举、bypass_confirmed）保留但标"二期预留、一期不产生"；口径统一为"仅红色异常 + 强校验兜底"。已改 AI预测与拆分 US-05 / Step-5 / 6.1 说明 / 6.2 / 6.3 / 7.1，及目标协同 AC-19。

## 3. 拆分权重配置的归属模块未明确

### 问题描述

`AI预测与拆分.md` 把“历史达成率 / 资金盘 / 阵地数”的权重配置定义为一期核心能力，且系统管理员应可维护。但 `目标协同管理.md` 的流程配置目前只覆盖：

- 是否拆解
- 拆解至层级
- 是否拆到机型

没有明确：

- 权重配置由哪个模块负责
- 配置入口在哪
- 配置变更如何生效
- 是否纳入审计日志与版本管理

### 涉及文档

- `docs/requirements/AI预测与拆分.md`
- `docs/requirements/目标协同管理.md`

### 建议

在需求阶段明确归属：

- 方案 A：权重配置归入 `目标协同管理` 的流程配置后台
- 方案 B：权重配置归入 `AI预测与拆分` 的独立规则配置能力

无论采用哪种方案，都应补齐配置入口、权限、审计和生效范围描述。

**✅ 结论（已闭环）**：采方案 B，权重配置归 AI 预测与拆分模块独立"拆分规则配置"能力。已在该模块新增 §4.4（入口/权限/审计/版本/生效范围）与 AC-08b，并在目标协同 6.1 注明流程配置不含权重。
