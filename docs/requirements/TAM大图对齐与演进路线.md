# TAM 大图对齐与演进路线

> 日期:2026-07-15
> 背景:收到公司级《目标与业绩管理 TAM 系统产品梳理》大图规划(15 个一级模块 K0–K15,规划周期数年)。本文记录我们当前"AI 目标管理平台(孟加拉一期)"在该大图中的定位、据此做出的一期地基调整(D-TAM),以及分阶段演进路线。
> 定位:一期不实现 TAM 全部模块;本文用于对齐战略纵深、指导地基预留、支撑竞选方案的演进叙事。

## 一、核心判断

TAM 大图与我们在建的平台**不是两套系统,而是同一系统的不同成熟度阶段**。我们当前建设的"目标拆解 → 追踪 → 预警"主干,约等于 TAM 大图 K3+K4+K7+K8+K9 的一期最小实现。用一句话概括覆盖度:**15 个一级模块中,我们深度覆盖 2 个、部分覆盖 6 个、暂未涉及 7 个。**

## 二、模块映射(我们当前系统 ↔ TAM K0–K15)

| TAM 模块 | 当前覆盖度 | 对应我们的能力 / 差距 |
|---------|-----------|---------------------|

## 三、D-TAM:一期地基调整(第2档预留,已落地设计)

大图最核心的主张是**目标模型从"单一销量"升级为"经营结果 + 产品结构 + 经营护栏"**,用以根治"国代压货达标、终端没增长的虚假繁荣"。我们一期只定销量目标,恰是该痛点的典型场景。

经评估选择**第 2 档:字段级预留,不做多指标功能**——成本接近于零,却把未来从"单指标"扩到"多指标"的改造从"改核心表结构"降级为"加数据行"。理由:目标指标扩展一旦发生,会波及计划/预测/拆分/达成的所有核心表与唯一约束,是全平台最贵的一类改造;而预留只需加一列。这与 D-4(规则包加 `rule_tier`、一期只用两级)是同一手法。

### D-TAM 落地清单(已改设计文档)

| 文档 | 表 / DTO | 改动 |
|------|---------|------|
| `design/目标协同管理.md` | `goal_wb_target_plan` | 加 `metric_code`(恒 `SALES_VOLUME`) |
| `design/目标协同管理.md` | `goal_wb_effective_key` | 加 `metric_code` + 纳入 `uk_effective` 唯一约束 |
| `design/AI预测与拆分.md` | `t_target_prediction` | 加 `metric_code`(预测按指标产出) |
| `design/执行追踪与复盘.md` | `t_execution_snapshot` | 加 `metric_code` + 纳入 `uk_snapshot` 唯一约束 |
| `design/数据接入与达成计算.md` | `achievement_summary` | 加 `metric_code` + 纳入 `uk_dim` 唯一约束 |
| `design/00-契约基线-接口清单.md` | `TargetPlanDTO`、`PredictionRequest` | 加 `metricCode`;查询类 DTO 一期不加(避免过度扩散) |

### D-TAM 关键设计原则

- `metric_code` 与 `target_type`(SI/SO/ST)**正交**:target_type 是"看哪个环节的量"(采购/终端/库存),metric_code 是"看什么指标"(一期只有销量)。
- 一期各模块对 metric_code **只透传、不分支**,不写任何多指标逻辑(严守 KISS,不为未做的需求写抽象)。
- 查询类 DTO(`AchievementQuery`/`EffectiveTargetQuery`/`GlobalMetricDTO`)一期不加 metric_code——单指标下无歧义;未来启用多指标时随该批查询统一补入。

## 四、演进路线(分阶段,非一次交付)

| 阶段 | 目标 | 覆盖 TAM 模块 |
|------|------|--------------|
| **一期(在建)** | 孟加拉试点:销量目标的拆解→追踪→预警闭环 | K3/K7 深度,K4/K6/K8/K12/K15 部分 |
| **近期候选** | ① 目标模型启用多指标(护栏:库存/价格,借 D-TAM 预留)② 轻量滚动预测(K8 空白,复用大数据能力)③ 执行期干预建议(D-7,K9 入口) | K1/K8/K9 补强 |
| **中期** | 审批流(接 BPM,承接 Demo 审批模板)· 激励联动(K10/K11,接返利数据)· 经营计划与资源(K5) | K4/K5/K10/K11 |
| **远期** | 经营诊断(K0)· 完整复盘归因(K12)· 会议节奏(K13)· 治理(K14)· 全球复制(规则包扩国家) | K0/K12/K13/K14 全量 |

> 路线为对齐方向,非承诺排期;每阶段范围以届时业务优先级与澄清为准。

## 五、由大图触发、待产品确认的候选项

1. **多指标目标是否近期启用**:D-TAM 已预留结构,是否要在近期把"库存/价格护栏"作为第 2、3 个指标真正落地?(功能增量,需求侧决策)
2. **滚动预测(K8)是否补齐**:从"看当前达成"升级为"预测月末能否达成",是业绩管理最有价值的能力之一,大数据预测能力可复用。是否排入近期?
3. **执行期干预建议(D-7 / K9 入口)**:上一轮已识别,大图将其放大为完整 K9 模块,印证其价值。优先级是否上调?

以上均不在一期范围内擅自展开,记录待产品排优先级。
