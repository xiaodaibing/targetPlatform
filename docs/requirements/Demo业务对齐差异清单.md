## 新发现:D-7(待产品确认,本轮未擅自新增设计)

> 完成 T-5 时发现:产品 Demo 中"AI 建议/处理建议"(如"Smart 8 DOS 52 天,建议下调目标并启用清尾策略")展示的是**执行期(月中看到异常时)**的即时建议;而当前设计中 `reviewAgent` 只在**周期结束后的复盘阶段**生成建议文本,执行追踪阶段的异常(AnomalyRecord)一期仅做规则打标,**无配套 LLM 建议生成**。

- **是否属于范围内**:D-6 原决策只授权"重写现有 Agent 的输出语言风格",不含"新增执行期实时建议生成"这一新能力——后者是功能增量,不是文本风格调整,故本轮未擅自设计,只在 `design/执行追踪与复盘.md` reviewAgent 小节标注了范围边界。
- **需要你拍板**:一期是否要补齐"执行期异常即时建议"?若要,需新增一个 Agent 调用点(在异常打标后触发,而非等复盘时),涉及 tracking 模块新增一次 LLM 调用与相应留痕/降级设计,工作量比文本重写大。若不做,Demo 里那类"看到异常立刻给动作建议"的体验,一期只能靠复盘阶段的滞后总结覆盖,需要在竞选方案里如实说明这个体验差距(或归入二期路线图)。
