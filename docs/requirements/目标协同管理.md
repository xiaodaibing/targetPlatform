# 需求规格：目标协同管理（PC 双工作台）

> 适用范围：一期孟加拉试点  
> 版本：v1.0  
> 日期：2026-07-02  
> 依据：`待澄清问题清单.md` 沟通结论回填区（两轮澄清已闭环，R1–R6 已确认）

---

## 模块目标

为目标创建者（国家经理/品牌经理）提供 PC 端双工作台，支持目标基础信息配置、AI 三档建议确认、拆分结果审核与直接发布生效，同时承载目标列表/风险/逾期视图、四级权限隔离、后端版本留痕及发布后通知触发。

---

## 范围边界

**做（一期）：**
- 目标管理工作台：全局指标、待办、目标列表、风险预警、流程逾期五视图
- 目标创建/拆解工作台：字段填写、总目标人工确认、拆分结果确认（正常批量 + 异常单独）、直接发布生效
- 目标创建字段规则（7.4 定义的所有字段，部分受一期约束处理）
- 流程配置：一期实际支持的配置项（拆解相关，审批相关不做）
- 版本留痕：仅后端存储，记录调整人/时间/原因，无前端版本对比 UI
- 四级权限隔离（国家/品牌/角色/经销商）与操作审计
- 发布后通知触发（调用 DCR APP 现有通知接口/事件，本平台仅触发）

**不做（一期，下方"明确不做项"详述）：**
- 审批流/加签/转交/退回（二期接 BPM）
- 对象移动端确认与申请调整
- 超时 Pass / 催办机制
- 返利关联（字段不展示）
- 前端版本对比 UI
- AI 对话助手
- 预测/拆分/强校验逻辑（属"AI 预测与拆分"模块）
- 执行追踪（属"执行追踪"模块）

| 角色 | 描述 | 本模块主要操作 |
|------|------|----------------|
| 国家经理 / 品牌经理 | 目标创建者，对全国或单品牌目标负责 | 创建目标、确认总目标、确认拆分、发布 |
| RSM（区域销售经理）| 对所辖区域目标负责 | 查看工作台（只读）；调整决策走线下沟通，系统内不提供 RSM 调整入口，调整结论由创建者/管理员经 §5.6 录入落账（P1 经 D-2 修订） |

> 孟加拉一期目标对象层级（R6）：国家 → 区域 → 目标对象（Dealer 客户 / 门店 / Promoter 人员）→ 机型

## 工作台功能

### 3.1 目标管理工作台

入口：PC 端首页，收敛为单一核心入口。

| 视图区域 | 展示内容 | 一期说明 |
|---------|----------|----------|
| 全局指标 | 当前周期所有目标的整体达成率汇总（按品牌/类型分组） | 数据来自执行追踪模块（跨模块依赖，见§13） |
| 待办 | 当前用户需处理的任务：总目标待确认、拆分结果待确认（含异常项待处理） | 仅展示当前用户权限范围内任务 |
| 目标列表 | 所有目标的概览列表：目标名称、类型、状态、周期、品牌；支持按类型/品牌/状态筛选 | 分页展示；可点击进入创建/拆解工作台 |
| 风险预警 | 红/黄/绿三色分级预警列表 | 执行阶段异常数据来自执行追踪模块；创建阶段异常来自 AI 拆分模块 |
| 流程逾期 | 目标周期开始日期前仍处于中间状态（草稿/总目标待确认/拆分结果待确认）的目标列表 | **一期仅展示，不触发催办或超时 Pass**；"逾期"定义为目标所属周期开始日 T 前未完成发布 |

### 3.2 目标创建/拆解工作台

分步操作界面，步骤顺序：

```
① 填写基础信息 → ② 确认总目标（AI 三档） → ③ 查看拆分结果并确认 → ④ 强校验 → ⑤ 发布生效
```

- 若配置为"不拆解"，跳过步骤③，总目标确认后直接执行步骤④⑤。
- 各步骤均展示当前目标的元信息（类型/品牌/周期）和进度导航。

## 目标创建字段规则

依据 7.4，以下为一期字段完整规则：

| 字段 | 必填 | 默认值 | 可选值 | 一期特殊处理 |
|------|------|--------|--------|-------------|
| 国家 | 是 | 若用户仅有单国权限则自动填充且禁用 | 预置国家列表（一期：孟加拉） | 单国权限用户不可切换 |
| 品牌 | 是 | 若用户仅有单品牌权限则自动填充且禁用 | TECNO / Infinix（一期）；**单次创建仅可选择一个品牌** | 多品牌权限用户可在授权品牌中任选其一，但单个目标仅绑定一个品牌 |
| 目标周期 | 是 | 当月（月度） | 月度 / 季度 / 自定义区间 | 自定义须满足边界条件（§9） |
| 目标类型 | 是 | 无 | SI / SO / ST | — |
| 目标对象 | 是 | 无 | Dealer / 门店 / Promoter | **一期不展示 Supervisor** |
| 品类与机型 | 否 | 不填则全品类 | 从产品库选择 | 不填时拆分模块按全品类处理 |
| 是否拆解 | 是 | 是 | 是 / 否 | 影响后续步骤是否进入拆解流程 |
| 拆解层级 | 否 | 按国家模板 | 区域 / 目标对象(Dealer/门店/Promoter) / 机型（孟加拉模板） | 选填；管理员可在流程配置中预设 |
| 是否审批 | — | 否（固定） | — | **一期固定为"否"，UI 展示但禁用，附提示"审批流将于二期上线"；不允许用户修改为"是"** |
| 是否关联返利 | — | — | — | **一期不展示此字段**（C6：返利不涉及） |

## 确认与发布流程

### 5.1 主流程

```
创建者填写基础信息
    ↓（系统校验必填字段）
目标进入"草稿"状态，系统向 AI 预测与拆分模块发送预测请求
    ↓（AI 预测就绪）
展示 AI 三档建议（保守/推荐/挑战）+ 推荐区间 + 置信度
    ↓
创建者通过滑动条/输入框确认总目标 → 点击"一键确认"
    ↓（版本快照：国家经理确认）
系统触发 AI 拆分（由 AI 预测与拆分模块执行）
    ↓（拆分结果就绪）
展示拆分结果（含异常标记）
    ↓
正常项：批量确认
异常项：逐条处理（填原因 / 修改值）
    ↓（所有项已处理）
系统执行强校验（下级合计 ≥ 上级，逐层校验）
    ↓（校验通过）
创建者点击"发布"→ 目标状态变为"已发布"→ 版本快照入库 → 触发通知
```

### 5.2 总目标确认交互

- AI 预测处于运行中（RUNNING）时，页面展示加载状态，步骤②禁用，不允许跳过等待。
- AI 预测完成（COMPLETED）时，页面展示三档建议、推荐区间和置信度，支持创建者通过滑动条/输入框确认总目标。
- AI 预测降级或失败（DEGRADED / FAILED）时，页面展示降级或失败原因提示，允许创建者手动直接输入总目标值；该人工兜底动作需留痕。
- 滑动条范围：以推荐区间为默认范围；用户可超出范围输入，超出时弹出提示"已超出 AI 推荐区间，请确认是否继续"，确认后可继续（调整原因记入版本）。

### 5.3 拆分结果确认交互

- 正常项标识：未触发任何异常的拆分结果行。
- 异常项标识（一期 3 类，与 AI 预测与拆分模块一致）：偏离 AI 区间（红）、偏离历史均值 > 50%（红）、下级合计不平衡（红）。
  > 异常类型来源于 AI 预测与拆分模块；本模块负责展示异常标记并收集处理结果。超市场容量/新品供应/清尾风险一期不做。
- 批量确认：仅对正常项生效，一次操作记录批次操作人和时间。
- 单条处理异常项：必须填写调整原因（字符数 ≥ 10，不可纯空格）或修改数值后才能标记为"已处理"。
- 所有异常项均标记为"已处理"后，"发布"按钮才解除禁用状态。

### 5.4 强校验规则（本模块触发，逻辑由 AI 预测与拆分模块提供）

| 校验维度 | 规则 | 违反后果 |
|---------|------|---------|
| 区域级 | 区域目标之和 ≥ 总目标 | 禁止发布，页面提示差额 |
| 对象级 | 下级对象目标之和 ≥ 区域目标 | 禁止发布，页面提示差额 |
| 机型级 | 机型目标之和 ≥ 对象目标（若拆到机型） | 禁止发布，页面提示差额 |

校验通过标准：所有层级均满足 `≥` 关系，0 容忍不通过。超额（下级合计 > 上级）允许。

### 5.5 发布

- 发布：直接生效，无任何审批卡点（R3 确认）。
- 发布后：目标状态 = 已发布，版本快照"最终发布版"入库，异步触发通知（§10）。
- 已发布目标的月中调整（**D-2 第三轮澄清修订**）：调整决策走线下沟通，**调整结果必须线上录入落账**——否则达成率分母失真。详见 §5.6。

### 5.6 发布后调整录入（D-2，一期新增）

> 背景：一期无审批、无 APP 端调整申请，但执行期市场变化必然产生目标调整；若调整只停留在线下，执行追踪模块的达成率分母即为错值，"目标达成率落 85%~110% 区间"这一核心业务指标全部失真。故一期支持**线下决策、线上落账**。

- **入口与权限**：仅目标创建者与系统管理员可对"已发布"目标发起调整录入；其余角色无入口（对象移动端调整申请仍属二期，见 R4/A3，本节不改变该边界）。
- **录入内容**：选择调整层级与对象 → 录入调整后目标值 → **调整原因必填**（线下沟通结论摘要）。
- **校验**：调整后全链路仍须满足强校验（下级合计 ≥ 上级，0 容忍）；违反时禁止提交并提示差额。
- **生效与留痕**：提交成功即生成"月中调整版"版本快照，目标**当前有效值**即时更新；执行追踪模块的达成率自下一计算周期起按当前有效值计算（见执行追踪 AC-23）。
- **通知**：调整生效后异步触发通知事件告知受影响对象（复用 §10 通道与可靠性要求）。

## 流程配置

### 6.1 一期实际支持的配置项

按"国家 / 品牌 / 目标类型"三维度配置，配置后新建目标自动应用：

| 配置项 | 说明 | 可配置值 |
|--------|------|----------|
| 是否拆解 | 控制目标创建流程是否包含拆解步骤 | 是 / 否 |
| 拆解至层级 | 拆解终止层级 | 区域 / 目标对象(Dealer/门店/Promoter) / 机型 |
| 是否拆到机型 | 在已配置拆解的场景下，是否进一步拆到机型维度 | 是 / 否 |

配置粒度：支持按国家+品牌+目标类型组合独立设置（如孟加拉+TECNO+SI 与孟加拉+Infinix+SI 可各自配置）。

> **不含拆分权重配置（方案 B，已确认）**：本流程配置只管"拆不拆/拆到哪层/是否拆到机型"的流程编排；拆分权重规则（历史达成率/资金盘/阵地数占比、平衡模式）属拆分算法参数，归 **AI 预测与拆分模块**的"拆分规则配置"能力维护，详见该模块 §4.4 / AC-08b，本模块不承载。

### 6.2 一期不支持的配置项（说明）

以下配置项存在于 7.10 原设计中，一期均不实现：

| 配置项 | 不做原因 |
|--------|----------|
| 是否需要审批 / 总目标审批 / 最终审批 | 审批流整体移至二期（接 BPM） |
| 是否需要区域确认 / 对象确认 / 财务审核 | 移动端确认整体移至二期 |
| 确认截止时间 / 超时处理规则 | 超时 Pass/催办机制不做（A3/C3） |
| 高风险加审规则 | 依赖审批流，二期 |
| 退回规则 | 依赖审批流，二期 |
| 审批人/确认人（按角色/按人员） | 依赖审批流，二期 |

## 版本留痕

### 7.1 版本快照节点

| 版本名称 | 触发时机 | 记录内容 |
|---------|----------|----------|
| AI 原始预测 | AI 三档建议就绪时 | AI 推荐值、置信度、推荐区间 |
| 国家经理确认 | 创建者点击"一键确认"总目标 | 确认值、确认人、时间 |
| AI 拆分结果 | AI 拆分完成时 | 各层级拆分值、异常标记 |
| RSM 调整 | 预留，一期不触发（RSM 无系统内调整入口；其线下调整结论经 §5.6 由创建者/管理员录入，落"月中调整版"） | — |
| 最终发布版 | 目标状态变为"已发布"时 | 全量各层级目标值 |
| 月中调整版 | 发布后调整录入提交成功时（§5.6，D-2 一期启用） | 调整层级/对象、调整前后值、调整原因、操作人、时间 |

### 7.2 实现约束

- **仅后端存储，一期无前端版本对比 UI**（C5 已确认）。
- 每条版本记录必含：操作人 ID、操作时间、版本类型、调整原因（AI 生成版本填"系统自动"）。
- 后端接口预留版本查询 API，供二期前端版本对比 UI 接入。

## 权限与审计

### 8.1 四级权限隔离

| 隔离层级 | 含义 | 实现要求 |
|---------|------|---------|
| 国家级 | 用户只能查看/操作其授权国家的目标数据 | 所有数据查询自动附加国家过滤条件 |
| 品牌级 | 用户只能查看/操作其授权品牌的数据 | 同上 |
| 角色级 | 不同角色的操作权限不同（创建/查看/发布等） | 按角色控制功能入口和操作按钮可见性 |
| 经销商级 | 经销商相关角色只能看到本经销商的目标数据 | 查询时附加 Dealer ID 过滤 |

越权请求：返回空集（非 403），不泄露数据存在与否。

### 8.2 操作审计

需要记录操作日志的关键动作：

| 动作 | 记录字段 |
|------|---------|
| 创建目标 | 操作人、时间、目标 ID、字段快照 |
| 确认总目标 | 操作人、时间、目标 ID、确认值、AI 推荐值 |
| 调整目标值（任意层级） | 操作人、时间、目标 ID、层级/对象、调整前值、调整后值、原因 |
| 批量确认拆分 | 操作人、时间、目标 ID、批次确认项数 |
| 单条处理异常项 | 操作人、时间、目标 ID、对象、处理方式、原因 |
| 发布 | 操作人、时间、目标 ID |
| 流程配置变更 | 操作人、时间、配置维度（国家/品牌/类型）、变更前值、变更后值 |

操作日志不可由普通用户删除或修改，支持管理员查询与导出（审计合规需求）。

## 边界条件

| 边界 | 规则 | 违反处理 |
|------|------|---------|
| 目标值范围 | 目标值 ≥ 0，不允许负数 | 输入框实时拦截，阻止提交 |
| 目标周期 | 最短 1 周，最长 12 个月；自定义区间起始日 ≥ 当前日期 | 提示具体约束，禁止提交 |
| 唯一生效目标 | 同一 国家/品牌/周期/类型/目标对象 维度组合只能有一个生效目标 | 发布时校验，重复则提示"该维度已存在生效目标"，阻止发布；数据库层唯一约束兜底（防并发） |
| 拆解层级 | 孟加拉模板最多 3 层（区域→目标对象→机型），不可配置超出范围 | 配置界面限制选项范围 |
| 审批人数（预留） | 单审批流最多 5 人——一期无审批，仅后端模型预留字段 | N/A（一期不校验） |
| 确认截止时间字段 | 不得早于当前时间，不得晚于目标周期结束时间（字段校验）——一期无超时执行逻辑 | 字段录入时校验，超出提示错误 |
| 数据延迟 | 外部系统数据延迟 ≤ 24 小时可接受；> 24 小时时工作台标记"数据异常" | 展示告警，不阻塞已有目标的操作 |

## 通知触发

### 10.1 触发时机

目标状态变更为"已发布"时，系统**异步**向 DCR APP 发出通知触发事件。

### 10.2 触发内容（最小集，详细契约由 architect 与 DCR APP 团队对齐）

- 目标 ID
- 品牌
- 国家
- 目标类型（SI/SO/ST）
- 目标对象范围（Dealer/门店/Promoter 列表 ID）
- 目标周期（起止日期）
- 发布时间

### 10.3 可靠性要求

- 触发结果（成功/失败/失败原因）记入通知触发日志。
- 触发失败不阻塞目标发布流程，目标状态仍为"已发布"。
- 支持管理员手动重试触发失败的通知。
- 具体触发接口形态（REST 回调 / Kafka 事件 / DCR APP SDK）为设计期依赖（R4，见§13）。

## 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | PC 端目标列表/工作台首屏加载 ≤ 2 秒（正常网络）；弱网延迟 ≤ 3 秒可接受 |

## 验收标准

### 工作台视图

- **AC-1** 目标管理工作台包含全局指标、待办、目标列表、风险预警、流程逾期五个区域，且所有区域的数据均仅展示当前用户权限范围内的内容，无权限数据不可见。
- **AC-2** 待办列表中显示当前用户待处理的"总目标待确认"和"拆分结果待确认"任务（含异常项待处理），点击可直接跳转至对应工作台的操作步骤。
- **AC-3** 目标列表支持按目标类型（SI/SO/ST）、品牌、目标状态至少三项筛选条件组合过滤，筛选结果无需刷新页面即实时更新。
- **AC-4** 风险预警视图中，红色预警目标与黄色预警目标以不同色标区分，管理者可在 30 秒内通过该视图识别当前所有高风险目标。
- **AC-5** 流程逾期视图展示"目标所属周期已开始但目标状态仍为草稿/总目标待确认/拆分结果待确认"的目标列表；一期该视图仅展示，不触发任何催办或超时 Pass 动作。

### 字段规则

- **AC-6** 仅拥有单个国家权限的用户创建目标时，国家字段自动填充为该国家且禁用，用户无法通过任何操作切换为其他国家。
- **AC-7** 仅拥有单个品牌权限的用户创建目标时，品牌字段自动填充且禁用；拥有多个品牌权限的用户可在授权品牌中选择其一创建目标，但单次创建只能选择一个品牌，不允许多选。
- **AC-8** 目标周期字段默认为当月月度；用户可切换为季度或自定义区间，三种方式的选择结果均正确映射为起止日期存储。
- **AC-9** 国家、品牌、目标周期、目标类型、目标对象、是否拆解六个必填字段，任一未填写时"下一步"按钮保持禁用状态，并在未填字段旁标红提示"必填"。
- **AC-10** 品类与机型字段为选填，用户未填写时系统不报错，且在后续拆分阶段自动按全品类处理。
- **AC-11** 目标对象下拉选项仅展示 Dealer、门店、Promoter 三项；Supervisor 不出现在选项列表中。
- **AC-12** "是否审批"字段在创建表单中展示，值固定为"否"且控件禁用，控件旁显示提示文本"审批流将于二期上线"；用户无法将其修改为"是"。
- **AC-13** 一期目标创建表单中不展示"是否关联返利"字段。

### 总目标确认

- **AC-14** 基础信息提交成功后，进入步骤②前若 AI 预测状态为 RUNNING，页面展示"AI 分析中，请稍候"加载状态，"一键确认"按钮禁用，不允许跳过等待。
- **AC-15** 当 AI 预测状态为 COMPLETED 时，页面同时展示保守值、推荐值、挑战值三档建议以及推荐区间和置信度；当 AI 预测状态为 DEGRADED 或 FAILED 时，页面展示降级或失败原因提示，并允许创建者手动输入总目标值，且该人工兜底动作写入操作日志。
- **AC-16** 滑动条与输入框联动同步，任一修改后另一个立即更新到相同值。
- **AC-17** 用户点击"一键确认"后，系统以当前值作为总目标触发 AI 拆分，确认动作写入操作日志（含确认值与 AI 推荐值），页面进入"拆分中"状态。

### 拆分结果确认与发布

- **AC-18** 拆分结果页面中，正常项（未触发异常标记的行）支持"批量确认"一次性操作；批量确认后所有正常项状态变更为"已确认"，并记录操作人和时间。
- **AC-19** 拆分结果页面中，异常项（一期均为红色标记，见 5.3）不能通过批量确认跳过；用户必须对每一个异常项逐条填写不少于 10 个字符的调整原因（或修改数值）后方可标记为"已处理"。
- **AC-20** 存在任意未处理的异常项时，"发布"按钮禁用；所有项均已确认或已处理后，"发布"按钮解除禁用。
- **AC-21** 强校验（下级合计 ≥ 上级）任意层级未通过时，"发布"按钮禁用，页面在对应层级高亮显示校验不通过信息及差额数值，不发起发布请求。
- **AC-22** 所有层级通过强校验且创建者点击"发布"后，目标状态立即变更为"已发布"，无任何审批卡点，整个操作在单次页面交互内完成。
- **AC-23** 配置为"不拆解"的目标类型，创建流程跳过拆解层级选择和拆分结果确认步骤，总目标确认后直接进入强校验和发布步骤。

### 流程配置

- **AC-24** 系统管理员可按"国家 + 品牌 + 目标类型"三维度组合设置"是否拆解"和"拆解至层级"，不同组合可独立配置，互不影响。
- **AC-25** 配置变更后，新建目标时系统自动应用对应模板的拆解层级默认值，无需创建者手动选择；配置变更写入操作日志。

### 版本留痕

- **AC-26** 以下五个节点系统自动创建版本快照并入库：AI 原始预测就绪、创建者确认总目标、AI 拆分结果生成、目标发布、发布后调整录入（§5.6）；每条快照记录包含操作人、操作时间、版本类型、调整原因。（RSM 在系统内无调整入口，其线下调整结论由创建者/管理员经 §5.6 录入落账——D-2 修订原 P1 口径）
- **AC-27** 一期不提供前端版本对比页面，所有版本快照仅存于数据库，且通过后端接口可查询（接口路径由架构师定义，供二期 UI 接入）。

### 权限与审计

- **AC-28** 跨权限的目标数据查询请求（如用户访问非授权国家的目标）返回空数据集，不返回 4xx 错误码，不暴露目标是否存在。
- **AC-29** 所有关键操作（创建、确认总目标、调整任意层级目标值、批量/单条确认拆分、发布、配置变更）自动写入操作日志，日志包含操作人 ID、操作时间、操作类型、目标 ID、调整前值、调整后值、原因字段。
- **AC-30** 操作日志不提供普通用户的删除/修改入口，支持管理员查询和导出。

### 边界条件

- **AC-31** 目标值输入框输入负数时，实时提示"目标值须 ≥ 0"并阻止表单提交。
- **AC-32** 自定义目标周期起始日期早于当前日期，或截止日期距起始日期超过 12 个月时，系统提示具体约束并禁止提交。
- **AC-33** 同一"国家/品牌/周期/类型/目标对象"组合已存在生效目标时，新目标的发布操作被阻止，返回提示"该维度已存在生效目标，不可重复发布"；并发创建场景由数据库唯一约束兜底保障。

### 通知触发

- **AC-34** 目标状态变更为"已发布"后，系统异步向 DCR APP 发出包含目标 ID、品牌、国家、对象范围、目标周期的通知触发事件；触发结果（成功/失败）写入通知触发日志。
- **AC-35** 通知触发失败时，目标状态保持"已发布"不受影响，失败原因记入日志，管理员可在后台手动重试触发。

### 非功能

- **AC-36** PC 端目标列表和工作台首屏在正常网络下加载时间 ≤ 2 秒（Lighthouse 性能测试），弱网（模拟延迟 500ms）下 ≤ 3 秒。
- **AC-37** 测试员从填写基础信息开始，到完成无异常拆分的发布操作，全流程操作时间 ≤ 5 分钟（操作计时测试）。
- **AC-38** Chrome 80+、Edge 80+ 浏览器下，1280×720 及以上分辨率，所有功能可正常使用。

### 发布后调整录入（D-2 新增）

- **AC-39** 目标创建者或系统管理员对"已发布"目标录入某层级对象的调整后目标值（调整原因必填）并提交成功后，系统生成"月中调整版"版本快照，且该对象的当前有效目标值即时更新为调整后值；其他角色在界面上无调整录入入口，直接调用接口返回权限拒绝。
- **AC-40** 调整录入提交时执行强校验：调整后任一层级出现"下级合计 < 上级"时，提交被阻止并提示具体差额；校验通过方可生效。
- **AC-41** 调整生效后，系统异步触发通知事件告知受影响对象（事件字段与 AC-34 一致，含调整前后值）；触发失败不影响调整已生效状态，失败处理同 AC-35。

## 依赖与风险

### 13.1 跨模块依赖（供 architect 定契约参考）

| 依赖方 | 依赖内容 | 优先级 | 备注 |
|--------|---------|--------|------|
| **AI 预测与拆分模块** | ① 预测请求/结果接口（三档建议、置信度、推荐区间）<br>② 拆分触发/结果接口（各层级拆分值、异常标记）<br>③ 强校验计算能力 | 阻塞 | 契约需在技术设计阶段与该模块对齐 |
| **执行追踪模块** | 全局指标（达成率汇总）、风险预警数据 | 非阻塞（工作台视图可先展示空数据） | — |
| **权限管理（支撑系统）** | 用户的国家/品牌/角色/经销商授权数据 | 阻塞 | 需确认权限数据 API 或共享数据来源 |
| **DCR APP 通知接口** | 通知触发接口/事件形态（REST 回调 / Kafka / SDK） | 非阻塞（发布流程独立，通知异步） | R4 设计期依赖，需 DCR APP 团队对齐 |

### 13.2 遗留设计依赖（R4 已登记，不阻塞需求）

- **通知触发方案（R4）**：一期仅触发，具体接口形态（REST 回调 / Kafka 事件 / DCR APP SDK）待 architect 与 DCR APP 团队在设计阶段对齐后确定。
- **审批预留（R3）**：二期审批走既有 BPM 系统，本模块需在数据模型中预留审批流程字段（如 `is_approval_required`、`approval_flow_id`），一期值为空/默认否。

### 13.3 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI 预测接口尚未就绪 | 步骤②无法展示三档建议，总目标确认受阻 | 降级方案：允许手动输入总目标；接口 Mock 先行开发 |
| DCR APP 通知接口未确定 | AC-34/AC-35 无法验证 | 通知触发异步解耦，本模块发布流程不依赖通知成功；接口形态待定前先记录触发意图到内部队列 |
| 数据体检缺位（R5） | AI 预测质量未知，置信度可能虚高 | 展示置信度让创建者感知；数据不足时强制降级提示 |
