## 1. 模块目标

为零售/品牌经理提供 PC 端工作台，承载**两条并行目标线**的全生命周期管理：

- **总目标线**：创建（手动输入 / 导入 / 调用 M02 测算）→ 调整 → **确认即生效，无审批**
- **人员目标线**：拆解（调用 M02）/ 批量导入 → 预检 → 提示性校验 → **送审（接公司 BPM）** → 审批通过发布生效 → 触发通知

以及版本留痕、操作审计、权限隔离、APP 端数据供给的接口出口。

## 2. 范围边界

### 做（一期）

- 目标管理工作台：全局指标、待办、总目标列表、人员目标列表、达成预警、流程逾期
- **总目标**：创建（手动输入 / 批量导入 / 触发 M02 测算）、列表、详情、调整、日志管理
- **人员目标**：拆解入口、批量导入、列表、详情、修改、修改日志
- **人员目标审批**：按国家 × 品牌 × 目标类型配置审批流模板，接公司 **BPM 系统**
- 版本留痕：仅后端存储，记录调整人/时间/原因/版本号，无前端版本对比 UI
- 权限隔离（国家 / 品牌 / 角色）与操作审计
- 发布后通知触发（调用 DCR APP 通知能力，本平台仅触发）
- **APP 端数据接口**：为 DCR APP 提供人员目标查询接口（页面由 DCR APP 团队实现）

### 不做（一期）

| 不做项 | 依据 |
|--------|------|
| 总目标审批 | E-1：总目标确认即生效 |
| 强校验（下级合计 ≥ 上级，0 容忍阻断） | E-7：改为 M02 的提示性校验 |
| 区域 / 门店 / Dealer 层级与对应工作台 | E-10 |
| AI 三档建议确认交互、置信度、AI 依据展示 | E-3 |
| 本平台自建移动端工程 | E-2：仅提供接口与交互设计 |
| 超时 Pass / 催办、加签 / 转交（由 BPM 侧能力决定，本平台不自建） | E-1 |
| 返利关联字段 | C6 |
| 前端版本对比 UI | 二期 |
| 复盘报告 | E-4 |
| 测算 / 拆解算法本身 | 属 M02 |
| 达成计算与预警判定 | 属 M01 / M04 |

## 3. 用户与角色

| 角色 | 本模块主要操作 |
|------|---------------|
| 零售经理 / 品牌经理 | 创建与调整总目标、发起人员目标拆解与导入、修改人员目标、提交审批 |
| 审批人（按 BPM 流程模板配置）| 在 BPM 侧审批人员目标；本平台展示审批状态与结论 |
| 促销员（BP）| 通过 DCR APP 查看本人目标与达成、接收预警（本平台仅提供接口）|

> 一期目标对象层级：**国家 → 人员（Promoter/BP）**，无区域/门店/Dealer 层（E-10）。

## 4. 工作台功能

### 4.1 目标管理工作台

| 视图区域 | 展示内容 | 说明 |
|---------|---------|------|
| 全局指标 | 当前周期总目标与人员目标合计、整体达成率 | 达成数据来自 M04（跨模块依赖）|
| 待办 | 当前用户待处理任务：总目标待确认、人员目标待提交、审批被驳回待修改 | 仅展示权限范围内任务 |
| 总目标列表 | 按时间 / 国家 / 品牌筛选；展示周期、品牌、总目标值、状态、机型覆盖率 | 分页；可进入详情 |
| 人员目标列表 | 按周期 / 品牌 / 门店 / BP 筛选；展示目标值、达成量、达成率、达成趋势、达成异常标记 | 达成相关列数据来自 M04 |
| 达成预警 | 达成预警与 N 天 0 销量异常列表 | 数据来自 M04 |
| 流程逾期 | 目标周期开始日前仍未生效的目标（总目标未确认 / 人员目标未审批通过）| **仅展示，不触发催办或超时 Pass** |

### 4.2 总目标流程

```
① 填写基础信息（国家/品牌/周期/类型）
   ↓
② 生成总目标值：三选一
   · 触发 M02 公式测算 → 展示基准值与计算明细 → 人工微调（原因必填）
   · 手动直接输入
   · 批量导入（模板含机型维度）
   ↓
③ 机型维度分解（M02 提供，人工可调；缺货机型受可信库存上限约束）
   ↓
④ 确认 → 总目标状态变为「已生效」（无审批卡点）→ 版本快照入库
```

### 4.3 人员目标流程

```
① 选择周期 / 品牌 / 目标类型（一期恒 SO）
   ↓
② 生成人员目标：二选一
   · 触发 M02 按国家规则包批量拆解（系数法，同时产出 SO 总量 + HSP 双目标）
   · 批量导入（模板含 SO 总量列与 HSP 列）
   ↓
③ 人工调整：逐条修改 / 批量编辑 / 重新导入覆盖（修改原因必填）
   ↓
④ 预检（M02 提供）→ 展示预计达成率区间与风险标记 R1/R2/R3（仅提示）
   ↓
⑤ 提交送审 → 触发提示性校验（M02 提供）
   · 合计 < 总目标 × 90%：弹窗展示缺口，用户可选「继续提交」→ 不阻断
   · 无同期总目标：不校验
   ↓
⑥ 发起 BPM 审批流程实例（按国家 × 品牌 × 类型匹配流程模板）
   ↓
⑦ 审批结果回调
   · 通过 → 状态「已发布」→ 版本快照入库 → 异步触发 DCR APP 通知
   · 驳回 → 状态「已驳回」→ 回到步骤③，驳回意见展示在页面
```

## 5. 字段规则

### 5.1 总目标创建字段

| 字段 | 必填 | 默认 | 可选值 | 一期处理 |
|------|------|------|--------|---------|
| 国家 | 是 | 单国权限自动填充且禁用 | 一期：孟加拉 | 单国权限不可切换 |
| 品牌 | 是 | 单品牌权限自动填充且禁用 | **一期仅 Infinix** | 单次创建仅绑定一个品牌 |
| 目标周期 | 是 | **下月**（月度）| 月度 / 自定义区间 | 自定义须展示起止日期并满足边界条件（§8）|
| 目标类型 | 是 | SO | **一期仅 SO**；SI / ST 为预留选项，选中时提示「暂未开放」 | — |
| 机型维度 | 否 | 全部 | 全部 / 中高端 / 新品 / 清尾（快捷选择）| 支持快速查看、批量导入、批量编辑、新增与删除机型 |
| 总目标值 | 是 | M02 测算基准值 | — | 人工修改时原因必填 |
| 调整原因 | 条件必填 | — | — | 值偏离测算基准值时必填（≥ 10 字符）|

> **已删除字段**：目标对象（一期总目标不选对象）、是否拆解、拆解层级（改由 M02 规则包的 `level_chain` 决定）、是否审批（总目标恒不审批）、是否关联返利。

### 5.2 人员目标字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 周期 / 品牌 / 目标类型 | 是 | 同总目标口径 |
| 目标对象 | 是 | **一期仅「人员」，单选**，不展示 Dealer / 门店 / Supervisor |
| BP / 所属门店 | 是 | 由 M02 拉取在职 BP 列表带出，不可手工新增不存在的人员 |
| SO 总量目标 | 是 | 值 ≥ 0；低于保底值 40 时提示 |
| HSP 目标 | 是 | 按规则包模式：`TOTAL` 填一个总数；`BY_MODEL` 逐机型填写，汇总只读 |
| 修改原因 | 条件必填 | 单条修改或批量编辑时必填（≥ 10 字符）|

## 6. 审批（E-1，接公司 BPM）

### 6.1 范围

- **总目标不走审批**，确认即生效。
- **人员目标必须审批**，审批能力由公司 BPM 系统承载，本平台不自建审批引擎。

### 6.2 配置

| 配置项 | 说明 |
|--------|------|
| 匹配维度 | 国家 × 品牌 × 目标类型 |
| 配置内容 | BPM 流程模板 ID、流程业务标识前缀、是否启用 |
| 未配置时 | 阻止提交并提示「该场景未配置审批流程，请联系管理员」|
| 权限 | 仅系统管理员可增改；变更写操作日志 |

> 审批人、审批层级、加签、转交、退回、超时策略**均由 BPM 侧流程模板定义**，本平台不重复建模、不提供对应配置项。

### 6.3 本平台职责边界

| 本平台做 | BPM 做 |
|---------|--------|
| 发起流程实例（携带目标计划 ID、周期、品牌、人员目标合计、缺口提示结果等业务参数）| 审批人路由、加签、转交、催办、超时 |
| 接收审批结果回调并流转本地状态 | 审批意见收集 |
| 展示当前审批状态、当前处理人、审批意见 | — |
| 提供审批详情跳转 BPM 的链接 | — |

### 6.4 状态机

```
DRAFT（草稿）
  → PENDING_APPROVAL（审批中）      // 提交成功、BPM 实例创建成功
  → APPROVED（审批通过）→ PUBLISHED（已发布，触发通知）
  → REJECTED（已驳回）→ DRAFT（修改后可再次提交）
  → SUBMIT_FAILED（提交失败）        // BPM 不可用，见 §6.5
```

### 6.5 BPM 不可用降级

- BPM 接口异常导致流程实例创建失败时，人员目标状态置为 `SUBMIT_FAILED`，**数据完整保留不丢失**，页面提示「审批系统暂不可用，请稍后重试」。
- 提供「重新提交」入口；管理员可查看提交失败列表并批量重试。
- 审批结果回调需**幂等**：同一流程实例的重复回调只生效一次。

## 7. 版本留痕

### 7.1 版本快照节点

| 版本类型 | 触发时机 | 记录内容 |
|---------|---------|---------|
| 总目标-测算 | M02 测算完成 | 基准值、公式编号、计算明细、数据可用性等级 |
| 总目标-确认 | 用户确认总目标 | 确认值、机型分解、确认人、时间、调整原因 |
| 总目标-调整 | 生效后调整提交成功 | 调整前后值、原因、操作人、时间、版本号 |
| 人员目标-生成 | M02 拆解完成 | 系数、各 BP 基础值与生成值、标记 |
| 人员目标-修改 | 单条修改 / 批量编辑 / 导入覆盖提交成功 | 调整前后值、原因、操作人、时间、版本号 |
| 人员目标-送审 | 提交送审成功 | 提交时的全量目标值、提示性校验结果（是否提示/缺口/用户选择）、BPM 实例 ID |
| 人员目标-发布 | 审批通过发布 | 全量目标值、审批人、审批意见、发布时间 |

### 7.2 实现约束

- **仅后端存储，一期无前端版本对比 UI**；后端提供版本查询接口供二期接入。
- 每条版本记录必含：操作人 ID、操作时间、版本类型、版本号、调整原因（系统自动生成的填「系统自动」）。
- 版本号在同一目标计划内单调递增。

## 8. 权限、审计与边界

### 8.1 权限隔离

| 层级 | 实现要求 |
|------|---------|
| 国家级 | 所有查询自动附加国家过滤 |
| 品牌级 | 所有查询自动附加品牌过滤 |
| 角色级 | 按角色控制功能入口与操作按钮可见性 |
| 人员级（APP）| BP 通过 APP 只能查询本人目标与达成 |

越权请求返回空集（非 403），不泄露数据存在与否。

> 原「经销商级隔离」一期无 Dealer 层，仅在数据模型保留字段，不实现过滤逻辑。

### 8.2 操作审计

需记录：创建总目标、确认总目标、调整总目标、机型分解调整、发起人员目标拆解、批量导入、修改人员目标、提交送审、审批结果回调、发布、通知触发与重试、审批流配置变更。
每条含：操作人 ID、时间、操作类型、目标计划 ID、对象标识、调整前值、调整后值、原因。日志不提供普通用户删改入口，支持管理员查询与导出。

### 8.3 边界条件

| 边界 | 规则 | 违反处理 |
|------|------|---------|
| 目标值范围 | ≥ 0，不允许负数；整数 | 输入框实时拦截 |
| 目标周期 | 最短 1 周，最长 12 个月；自定义区间起始日 ≥ 当前日期 | 提示约束，禁止提交 |
| 唯一生效目标 | 同一「国家 / 品牌 / 周期 / 类型 / metric_code」组合只能有一个生效**总目标**；同一「国家 / 品牌 / 周期 / 类型 / BP / metric_code / 机型」组合只能有一条生效**人员目标** | 发布时校验并提示；数据库唯一约束兜底防并发 |
| 数据延迟 | 外部数据延迟 ≤ 24 小时可接受；> 24 小时工作台标记「数据异常」 | 展示告警，不阻塞已有操作 |
| 批量导入 | 单次 ≤ 1000 行；模板校验失败整批回滚并输出错误行清单 | 不做部分导入 |

## 9. 通知触发与 APP 接口

### 9.1 通知触发

| 时机 | 接收方 | 内容最小集 |
|------|-------|-----------|
| 人员目标审批通过并发布 | 对应 BP | 目标计划 ID、品牌、国家、周期起止、SO 总量目标、HSP 目标、发布时间 |
| 人员目标发布后被修改并重新发布 | 受影响 BP | 同上 + 调整前后值 |
| 审批被驳回 | 提交人 | 目标计划 ID、驳回意见、驳回人、时间 |

- 通知为**异步触发**，触发结果（成功/失败/原因）记入通知触发日志。
- 触发失败不阻塞目标发布，状态仍为「已发布」；管理员可手动重试。
- 具体触发形态（REST 回调 / Kafka / DCR APP SDK）为设计期依赖，需与 DCR APP 团队对齐。

### 9.2 APP 端接口供给（E-2）

本平台**不建移动端工程**，仅向 DCR APP 提供接口与交互设计。一期需提供：

| 接口 | 用途 | 数据来源 |
|------|------|---------|
| 本人目标概览（首页卡片）| 当前周期 SO 总量目标、HSP 目标、达成量、达成率、剩余天数 | M03 + M04 |
| 本人目标详情 | 目标构成、HSP 明细（`BY_MODEL` 时含机型）、发布时间、当前版本 | M03 |
| 本人达成概览与销售明细 | 达成趋势、按日/按机型销售明细 | M04 |
| 预警消息列表与已读/处理 | 达成预警、N 天 0 销量异常 | M04 |

接口须做人员级权限校验：BP 只能查询本人数据。

## 11. 验收标准

### 工作台

- **AC-01** 工作台包含全局指标、待办、总目标列表、人员目标列表、达成预警、流程逾期六个区域，所有区域仅展示当前用户权限范围内数据。
- **AC-02** 总目标列表支持按时间、国家、品牌三项筛选组合过滤，结果实时更新无需刷新页面。
- **AC-03** 人员目标列表展示目标值、达成量、达成率、达成趋势与达成异常标记，并支持按周期/品牌/门店/BP 筛选。
- **AC-04** 流程逾期视图展示「周期已开始但总目标未生效或人员目标未审批通过」的记录；仅展示，不触发任何催办或超时 Pass 动作。

### 总目标

- **AC-05** 用户可通过「触发测算」「手动输入」「批量导入」三种方式生成总目标值，三种方式产生的记录均可进入机型分解与确认步骤。
- **AC-06** 总目标值偏离 M02 测算基准值时，调整原因为必填（≥ 10 字符且非纯空格），未填写则确认按钮禁用。
- **AC-07** 用户点击「确认」后总目标状态立即变为「已生效」，**不经过任何审批卡点**，操作在单次页面交互内完成，并生成「总目标-确认」版本快照。
- **AC-08** 总目标生效后仍可调整；调整提交成功生成「总目标-调整」版本快照，记录调整人、时间、版本号、调整前后值与原因，当前有效值即时更新。
- **AC-09** 总目标详情页可查看创建详情（基准值、公式、计算明细、机型分解、各次调整记录）。
- **AC-10** 目标周期字段默认为**下月**；切换为自定义区间时页面展示开始日期与结束日期两个输入项。
- **AC-11** 目标类型下拉中 SI、ST 为不可选或选中即提示「暂未开放」，一期只能创建 SO 类型目标。
- **AC-12** 品牌下拉仅展示当前用户被授权的品牌；一期仅 Infinix；单次创建只能选择一个品牌。

### 人员目标

- **AC-13** 用户可通过「触发拆解」或「批量导入」生成人员目标；两种方式均同时产生 SO 总量目标与 HSP 目标两类记录。
- **AC-14** 目标对象下拉仅展示「人员」一项，且为单选；Dealer、门店、Supervisor 不出现在选项中。
- **AC-15** 单条修改或批量编辑人员目标时，修改原因为必填（≥ 10 字符），未填写则保存按钮禁用；保存成功生成「人员目标-修改」版本快照，含操作人、时间、版本号。
- **AC-16** 人员目标详情页可查看该 BP 的 SO 总量目标、HSP 目标（`BY_MODEL` 模式下含机型明细）、基础值、系数、生成值、各类标记与修改历史。
- **AC-17** 批量导入超过 1000 行或模板校验失败时，整批不入库并输出具体错误行号与错误原因清单。

### 审批（E-1）

- **AC-18** 人员目标提交送审时，系统按「国家 × 品牌 × 目标类型」匹配 BPM 流程模板并创建流程实例；未配置模板时阻止提交并提示「该场景未配置审批流程，请联系管理员」。
- **AC-19** 提交成功后人员目标状态变为「审批中」，页面展示当前审批状态、当前处理人与跳转 BPM 的链接；此状态下不可编辑目标值。
- **AC-20** BPM 回调审批通过时，状态变为「已发布」并生成「人员目标-发布」版本快照，同时异步触发通知；回调驳回时状态变为「已驳回」，驳回意见展示在页面，用户修改后可再次提交。
- **AC-21** 同一 BPM 流程实例的重复回调只生效一次（幂等），不产生重复版本快照或重复通知。
- **AC-22** BPM 接口异常导致流程实例创建失败时，人员目标数据完整保留、状态为「提交失败」，页面提示「审批系统暂不可用，请稍后重试」，且提供重新提交入口。
- **AC-23** 总目标在任何路径下都不进入审批流程，系统不存在总目标送审的入口或接口。

### 提示性校验（与 M02 协同）

- **AC-24** 人员目标合计低于同期总目标 90% 时，提交时弹出提示展示缺口数值与占比，用户点击「继续提交」后正常进入审批流；提交按钮在任何情况下不被禁用。
- **AC-25** 系统中不存在任何因「下级合计 < 上级」而阻断提交或发布的路径（E-7）。

### 版本与审计

- **AC-26** §7.1 的七类版本快照在对应节点自动入库，每条含操作人、时间、版本类型、版本号、调整原因。
- **AC-27** 一期不提供前端版本对比页面；版本数据可通过后端接口查询。
- **AC-28** §8.2 列出的关键操作全部写入操作日志，含操作人 ID、时间、操作类型、目标计划 ID、对象标识、调整前后值、原因；日志无普通用户删改入口，管理员可查询导出。

### 权限与边界

- **AC-29** 跨权限的目标数据查询返回空数据集，不返回 4xx，不暴露数据是否存在。
- **AC-30** 目标值输入负数或小数时实时提示并阻止提交。
- **AC-31** 同一「国家/品牌/周期/类型/metric_code」已存在生效总目标时，重复生效被阻止并提示；并发场景由数据库唯一约束兜底。
- **AC-32** 自定义周期起始日早于当前日期，或跨度超过 12 个月时，提示具体约束并禁止提交。

### 通知与 APP 接口

- **AC-33** 人员目标发布后异步触发通知事件，含目标计划 ID、品牌、国家、周期、SO 总量目标、HSP 目标、发布时间；触发结果写入通知日志。
- **AC-34** 通知触发失败时目标状态保持「已发布」，失败原因记入日志，管理员可手动重试。
- **AC-35** §9.2 四类 APP 接口均实现人员级权限校验：BP 请求他人数据时返回空数据集。

### 非功能

- **AC-36** PC 工作台首屏正常网络 ≤ 2 秒，弱网（模拟 500ms 延迟）≤ 3 秒。
- **AC-37** 无异常场景下，从创建总目标到人员目标提交送审全流程操作时间 ≤ 10 分钟。
- **AC-38** Chrome 80+、Edge 80+，1280×720 及以上分辨率下所有功能可正常使用。

## 12. 依赖与风险

### 12.1 跨模块依赖

| 依赖方 | 内容 | 阻塞 |
|--------|------|------|
| **M02 目标测算与拆解** | 总目标测算与机型分解、人员目标批量拆解、预检结果、提示性校验结果 | 是 |
| **M04 执行追踪与预警** | 达成量、达成率、达成趋势、预警数据（工作台与 APP 接口使用）| 否（可先展示空数据）|
| **M01 数据接入** | 数据延迟状态（用于「数据异常」标记）| 否 |
| **公司 BPM 系统**（外部）| 流程实例创建、审批状态查询、结果回调 | **是**（AC-18~22 依赖）|
| **DCR APP 团队**（外部）| 通知触发形态、APP 页面实现与接口联调 | 否（发布流程不依赖通知成功），但影响 APP 端交付 |
| **权限管理**（外部）| 用户的国家/品牌/角色授权数据 | 是 |

### 12.2 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| BPM 流程模板与接口契约未定 | 人员目标全流程无法闭环 | 尽早对齐；先以 Mock 开发；保留 `SUBMIT_FAILED` 降级路径保证数据不丢 |
| DCR APP 团队排期不可控 | APP 端能力一期无法交付 | 接口先行、契约先定；PC 端闭环独立可用，APP 缺失不阻塞一期上线 |
| 审批引入后目标下发时点变晚 | 与「当月 5 号前下发」的业务诉求冲突 | 流程模板尽量单级；工作台流程逾期视图暴露卡点 |
