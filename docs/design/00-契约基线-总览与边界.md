```

│   ├── facade       对外暴露：AchievementQueryFacade / HistoryDataQueryFacade / DataSourceStatusFacade
│   ├── feign        对外部系统的 Feign：DrpClient / DcrClient / ActivationClient / InventoryClient / ProductClient / HolidayClient
│   ├── job / service / mapper / entity …
├── forecast        AI 预测与拆分（M02）
│   ├── facade       PredictionFacade / SplitFacade
│   ├── feign        BigDataForecastClient（大数据团队实时基准）
│   ├── tool         AgentScope 工具（@Component）：ForecastBaselineTool / HistoryStatTool …
│   ├── service（三档规则引擎 / 拆分引擎 / 平衡 / 强校验 / 异常识别）…
├── workbench       目标协同管理（M03，PC 双工作台）
│   ├── facade       TargetPlanFacade / EffectiveTargetQueryFacade
│   ├── feign        DcrAppNotifyClient / UacAuthClient
│   ├── controller（REST）/ service …
├── tracking        执行追踪与复盘（M04）
│   ├── facade       GlobalMetricsFacade
│   ├── feign        DcrAppNotifyClient（共享定义见 §5）/ BigDataFeedbackClient
│   ├── tool         AgentScope 工具（@Component）：ReviewStatTool …
│   ├── job / service（达成率/异常识别/Gap/复盘/回流）…
└── agent           AgentScope 集成层（共享）
    ├── config       Model / Toolkit / Agent 单例 Bean；ModelRegistry（Provider 按 ADR-0004 选型）
```

> 说明：AgentScope 业务工具（@Component）就近放在其**业务模块**内（forecast / tracking），由 `agent` 层的 `Toolkit` Bean 统一 `registerTool`；`agent` 层只放共享基座与横切能力（脱敏、审计、线程池）。

## 3. 调用边界规则（硬约束，ArchUnit 守卫）

| 场景 | 允许方式 | 禁止 |
|------|---------|------|
| 应用内模块间协作 | 依赖对方 `facade` 接口（Spring 同容器注入）；或订阅对方发布的 Kafka 事件 | ❌ 直连对方表/Mapper；❌ 用 Feign 调本应用内部模块；❌ 依赖对方 `service/entity` 内部类 |
| 对外部系统 | `feign` 包内 OpenFeign 声明式客户端（经 Eureka 发现） | ❌ 手写 HTTP/RestTemplate 直连外部；❌ 直连外部数据库 |
| 共享契约 | 只放 `common` 包；facade 入参/出参、Kafka 事件体、对外 Feign 的 DTO 均取自 common | ❌ 在模块内重复定义同一 DTO；❌ 让 common 依赖任何业务模块 |
| AI 能力 | 经 `agent` 层 Agent Bean 调用；业务工具走 `Toolkit.registerTool` | ❌ 业务模块裸调 LLM SDK；❌ 绕过脱敏中间件发数据出境 |

依赖方向（单向，禁止环）：`common ← 各业务模块 ← agent`；`common` 不依赖任何模块；业务模块间只经 facade 接口（编译期都依赖接口，接口与 DTO 在被调模块 facade 包 + common）。

> 落地守卫：用 ArchUnit 规则断言「`goal.<A>.service..` 不得依赖 `goal.<B>.service..`/`..mapper..`」「`goal.common..` 不依赖 `goal.<module>..`」「`feign` 包注解 `@FeignClient` 仅出现在对外客户端」。

## 5. 共享契约目录（common 包）

### 5.1 核心枚举（`common.enums`）

| 枚举 | 取值 | 使用方 |
|------|------|--------|
| `TargetType` | SI / SO / ST | 全模块 |
| `Brand` | TECNO / INFINIX | 全模块（一期仅二品牌） |
| `ObjectLevel` | COUNTRY / REGION / DEALER / SHOP / PROMOTER / MODEL | 拆分/追踪层级 |
| `ObjectType` | DEALER / SHOP / PROMOTER | 达成计算对象维度 |
| `AvailabilityLevel` | FULL / PARTIAL / LIMITED | 数据可用性 / 预测数据充分性 |
| `Granularity` | DAY / WEEK / MONTH | 追踪时间粒度 |
| `Severity` | RED / YELLOW / GREEN | 状态色 / 异常级别 |
| `AnomalyType` | 拆分期：INTERVAL_DEVIATION / HISTORY_DEVIATION / TOTAL_IMBALANCE；执行期：LOW_ACHIEVEMENT / SLOW_PROGRESS / ABNORMAL_CHANGE / HIGH_DOS | forecast / tracking |
| `PredictionStatus` | PENDING / RUNNING / COMPLETED / DEGRADED / FAILED | forecast |
| `SplitStatus` | PENDING / SPLITTING / DRAFT / ADJUSTING / VALIDATION_PASS / VALIDATION_FAIL | forecast |
| `TargetPlanStatus` | DRAFT / TOTAL_PENDING / SPLIT_PENDING / PUBLISHED | workbench |
| `NotificationType` | TARGET_PUBLISHED / ALERT_RED / ALERT_YELLOW / REVIEW_GENERATED | workbench / tracking |
| `BalanceMode` | AI_WEIGHT / HISTORY / RATE / AVERAGE | forecast |

> 国家一期固定孟加拉，用 `countryCode`（ISO，如 `BD`）字符串字段承载，不做枚举以留全球复制扩展位。

### 5.2 统一错误码（`common.error.GoalErrorCode`）

按模块前缀编排，`GoalBizException(GoalErrorCode, args...)` 抛出，全局异常处理器转标准响应体（沿用 Hulk 统一返回结构 + 国际化 message key）。

| 错误码 | 触发 | 语义（非空返回，禁止 null/0/缓存兜底） |
|--------|------|------|
| `DATA_LIMITED` | DCR 等核心源受限 | 达成/查询接口返回该码，响应体不含达成数字 |
| `RULE_NOT_DEFINED` | ST/门店/人员 等"无此规则"维度 | 不计算不返回 0 |
| `HISTORY_INSUFFICIENT` | 历史跨度 < 12 个月 | 附带当前可用最大跨度天数 |
| `PREDICTION_FAILED` | 预测服务异常/数据完全不可用 | 允许人工设定降级 |
| `SPLIT_NO_RULE` | 该国家/品牌/类型无拆分规则配置 | 阻止自动拆分 |
| `VALIDATION_FAILED` | 强校验不通过 | 附带层级名 + 差额 |
| `LOCK_CONFLICT` | 锁定值之和 ≥ 上级 | 禁止平衡/提交 |
| `DUPLICATE_EFFECTIVE_TARGET` | 同维度已存在生效目标 | 阻止发布（DB 唯一约束兜底） |
| `NOTIFY_FAILED` | 通知触发失败 | 不阻塞主流程，记日志，可重试 |

> 权限越权统一策略（AC-28/AC-21 对齐）：数据查询类**返回空集**，不返回 4xx，不暴露数据是否存在。此为跨模块统一约定，非错误码。

### 5.3 Kafka 主题（应用内异步解耦，命名 `goal.<producer>.<event>`）

> **事件信封（所有事件体统一）**：`eventId`（UUID，消费方去重）、`eventTime`。

| 主题 | 生产者 | 消费者 | 事件体（common.event，权威字段） | 幂等键 |
|------|--------|--------|------------------------|--------|
| `goal.dataingest.achievement-updated` | dataingest | tracking | `AchievementUpdatedEvent`：countryCode, brandCode, targetType, objectType, objectId, periodMonth(YYYYMM), achievementValue, dataQualityFlag(AvailabilityLevel), calcTime, calcBatchId ——**逐对象/逐维度发**（每维度一条，非批次汇总） | (objectId,targetType,periodMonth)；消费方另用 `eventId` 去重 |
| `goal.dataingest.data-source-anomaly` | dataingest | workbench / 运维 | `DataSourceAnomalyEvent`：sourceId, sourceName, lastSuccessTime, delayHours, availabilityLevel, consecutiveFailures, errorMessage | (sourceId,日期) |
| `goal.dataingest.data-ready` | dataingest | forecast / tracking（触发下游批量） | `DataReadyEvent`：sourceId, sourceType, countryCode, brandCode?, pullType, cursorFrom, cursorTo, pullCount, batchId | (sourceId,日期) |

> **通知类事件（发往 DCR APP）**：R4 待与 DCR APP 团队对齐形态（REST 回调 / Kafka / SDK）。契约基线先以 `DcrAppNotifyClient`（Feign）定义（见 §接口清单），若最终定为 Kafka 再切换实现，事件体 `NotificationEventDTO` 不变。

## 7. 非功能与横切（基线约定，各模块设计沿用）

- **权限四级隔离（国家/品牌/角色/经销商）**：统一在 facade/Controller 入口按 `RuntimeUser`（来自 UAC）附加过滤条件；越权返回空集。权限数据来源见接口清单 `UacAuthClient`。
- **审计留痕**：关键操作（创建/确认/调整/发布/异常处置/通知/回流）统一走 `OperationLog`，字段：操作人ID/时间/类型/目标ID/前值/后值/原因；普通用户不可删改。
- **幂等**：外部拉取按 (数据源,日期范围) 幂等；达成事件按 §5.3 幂等键；通知按 (对象ID,异常类型,自然日) 幂等。
- **国际化**：错误 message、业务依据/复盘文本支持中/英（`Accept-Language` 驱动）。
- **可观测性**：AgentScope 链路挂 `OtelTracingMiddleware`；外部 Feign、Kafka 消费、定时 Job 统一埋点。
- **配置外部化**：Apollo；LLM Key、外部系统地址、三档规则/异常阈值走配置中心，严禁硬编码。
