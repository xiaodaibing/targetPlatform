# 技术设计:契约基线（二）接口清单

> 版本：v1.0 | 日期：2026-07-03 | 作者：architect
> 定位：gate #1 产物。模块**门面接口**（应用内协作）+ **对外 Feign 接口**（外部系统）+ 核心 DTO 字段。
> 约束：门面接口入参/出参 DTO 均在 `common.dto`；Feign 仅用于外部系统；签名为契约，实现不得偏离。

---

## 一、模块门面接口（`com.transsion.goal.<module>.facade`）

> 均为 Spring 同容器注入的 Java 接口，非 REST。返回值省略 Hulk 统一包装，只描述业务载荷。异常经 `GoalBizException(GoalErrorCode)` 抛出。

### 1. dataingest —— `AchievementQueryFacade`

```java
public interface AchievementQueryFacade {
    /** 查询单对象某周期 SI/SO/ST 达成汇总；核心源受限时抛 DATA_LIMITED；无规则维度抛 RULE_NOT_DEFINED */
    AchievementDTO getAchievement(AchievementQuery query);

    /** 批量查询（追踪看板/Gap 下钻用），按对象列表返回；越权对象过滤为空 */
    List<AchievementDTO> listAchievements(AchievementBatchQuery query);
}
```
`AchievementQuery`：countryCode, brand, targetType, objectType, objectId, periodMonth(YYYYMM)。
`AchievementDTO`：上述维度 + achievementValue(BigDecimal) + dataQualityFlag(AvailabilityLevel) + dataCutoffDate + calcTime。

### 2. dataingest —— `HistoryDataQueryFacade`

```java
public interface HistoryDataQueryFacade {
    /** 历史明细查询（预测/复盘/对比）；跨度<12月抛 HISTORY_INSUFFICIENT（附最大可用跨度天数） */
    HistoryDataDTO queryHistory(HistoryQuery query);

    /** 数据充分性检查（预测前置 Step-1），返回等级 + 跨度，不抛异常 */
    DataAvailabilityDTO checkAvailability(AvailabilityQuery query);
}
```
`HistoryQuery`：countryCode, brand, dataType(DRP/DCR/ACTIVATION/INVENTORY), fromDate, toDate。
`HistoryDataDTO`：records + spanDays + availabilityLevel。
`DataAvailabilityDTO`：availabilityLevel + spanMonths + missingFields。

### 3. dataingest —— `DataSourceStatusFacade`

```java
public interface DataSourceStatusFacade {
    /** 各数据源当前可用性等级与最近更新时间（运维/工作台"数据异常"标记用） */
    List<DataSourceStatusDTO> listStatus(String sourceId /* nullable=全部 */);
}
```
`DataSourceStatusDTO`：sourceId, sourceName, availabilityLevel, lastSuccessTime, consecutiveFailures, isAnomaly, lastErrorMessage。

### 4. forecast —— `PredictionFacade`

```java
public interface PredictionFacade {
    /** 触发预测：调大数据基准→规则生成三档→LLM 业务依据(HITL)；异步，返回受理句柄 */
    PredictionHandle predict(PredictionRequest request);

    /** 查询预测结果（前端轮询；状态 PENDING/RUNNING/COMPLETED/DEGRADED/FAILED） */
    PredictionResultDTO getPrediction(Long targetPlanId);
}
```
`PredictionRequest`：targetPlanId, countryCode, brand, targetType, metricCode, targetPeriod, objectType（可选机型范围）。
`PredictionResultDTO`：conservativeValue / recommendValue / challengeValue / intervalLower / intervalUpper / confidence(0-100) / basisText(中英) / predictionSource / dataAvailability / status。
> 约束（AC-05）：保守≤推荐≤挑战，下限≤推荐≤上限；违反则整体 FAILED，六字段全空或全有，禁止部分输出。

### 5. forecast —— `SplitFacade`

```java
public interface SplitFacade {
    /** 接确认后的总目标，按规则逐层拆分（国家→区域→Dealer→门店→Promoter→机型），返回初稿+异常 */
    SplitResultDTO split(SplitRequest request);         // 无规则配置抛 SPLIT_NO_RULE

    /** 人工修改某层值后触发自动平衡（锁定值不动）；锁定和≥上级抛 LOCK_CONFLICT */
    SplitResultDTO rebalance(RebalanceRequest request);

    /** 强校验（CV-1/2/3，≥ 且 0 容忍）；不过抛 VALIDATION_FAILED（附层级名+差额） */
    ValidationResultDTO validate(Long targetPlanId);

    /** 拆分结果查询（工作台确认页展示） */
    SplitResultDTO getSplitResult(Long targetPlanId);
}
```
`SplitRequest`：targetPlanId, confirmedTotal(BigDecimal), balanceMode。
`RebalanceRequest`：targetPlanId, changedLevel, changedDimensionId, newValue, balanceMode。
`SplitResultDTO`：targetPlanId, status(SplitStatus), items[]，其中每 item：level(ObjectLevel), dimensionId, dimensionName, allocatedValue, isLocked, isManual, adjustReason, version, anomalies[]。
`Anomaly`：anomalyType, severity(拆分期一期恒 RED；YELLOW 二期预留), triggerDesc, isResolved, resolutionNote, bypassConfirmed(二期预留)。
`ValidationResultDTO`：passed(bool), failures[]（level, name, subtotal, parentValue, gap）。

### 6. workbench —— `TargetPlanFacade`

```java
public interface TargetPlanFacade {
    /** 供 forecast 回传"确认后的总目标"后读取计划元信息（品牌/国家/类型/周期/是否拆解/拆解层级） */
    TargetPlanDTO getPlan(Long targetPlanId);
}
```
`TargetPlanDTO`：id, countryCode, brands[], targetPeriod(起止), targetType, metricCode, objectType, splitEnabled, splitToLevel, splitToModel, status。

> **metricCode（D-TAM 第2档预留）**：目标指标编码，一期恒 `SALES_VOLUME`。仅在契约源头 `TargetPlanDTO` 与下方 `PredictionRequest` 携带，供未来多指标（份额/利润/结构占比，对齐 TAM 大图 K1）扩展；一期各模块透传即可、不做分支逻辑。查询类 DTO（`AchievementQuery`/`EffectiveTargetQuery`/`GlobalMetricDTO` 等）**一期不加** metricCode——一期单指标下它们无歧义，避免字段过度扩散；未来启用多指标时随该批查询一并补入。

### 7. workbench —— `EffectiveTargetQueryFacade`

```java
public interface EffectiveTargetQueryFacade {
    /** 供 tracking 取"已生效目标"（目标量/周期/对象层级），目标生效后才追踪 */
    List<EffectiveTargetDTO> listEffectiveTargets(EffectiveTargetQuery query);
}
```
`EffectiveTargetQuery`：countryCode, brand, targetType, periodMonth（可选对象层级/ID）。
`EffectiveTargetDTO`：targetPlanId, objectLevel, objectId, objectName, targetQty, periodStart, periodEnd。

### 8. tracking —— `GlobalMetricsFacade`

```java
public interface GlobalMetricsFacade {
    /** 工作台全局指标：按品牌/类型分组的整体达成率汇总 */
    List<GlobalMetricDTO> globalAchievement(MetricQuery query);
    /** 工作台风险预警：红/黄分级预警列表 */
    List<RiskAlertDTO> riskAlerts(MetricQuery query);
}
```
`GlobalMetricDTO`：brand, targetType, periodMonth, avgAchievementRate, greenCount, yellowCount, redCount。
`RiskAlertDTO`：objectLevel, objectId, objectName, severity, anomalyType, achievementRate, gapValue。

## 二、对外 Feign 接口（`com.transsion.goal.<module>.feign`，经 Eureka 发现）

> 均标 `@FeignClient`；真实路径/入参/认证方式**待外部团队确认**（见总览 §8 遗留项）。此处定契约结构与 DTO，占位方法在对齐后回填。统一：超时 ≥30s、指数退避重试≤3、失败降级见各模块需求。

### dataingest 的外部客户端

| Feign 客户端 | 外部系统 | 关键方法（占位） | 用途 | 遗留 |
|-------------|---------|-----------------|------|------|
| `DcrClient` | DCR | `imeiSalesQuery(...)` / `imeiPurchaseQuery(...)` / `shopSalesQuery(...)` | SI/SO/ST 取数核心 | **R2**：ST vs SO(Dealer) 差异口径钉死 |
| `DrpClient` | DRP | `pullDelivery(cursor)` / `capitalPool(dealerIds)` | SI 进货流向 + 资金盘 | **OQ-02** 资金盘归属 |
| `ActivationClient` | UAC 激活 | `pullActivation(range)` | 激活辅助维度 | — |
| `InventoryClient` | 库存系统 | `pullInventory(range)` / `dos(objectIds)` | 库存/DOS | OQ-4 DOS 计算归属 |
| `ProductClient` | 产品库 | `listModels(brand)` | 机型/品类映射 | — |
| `HolidayClient` | 假期日历 | `listHolidays(countryCode, year)` | 工作日/周期调整 | **R-NEW-02** 来源（配置中心 or 外部） |
| `TerritoryClient` | DCR 人员/门店 | `dealerCount / shopCount / promoterCount(...)` | 阵地数（拆分权重） | **OQ-03** |

> 拉取机制统一：REST 定时拉取（禁直连库）；游标增量（Delivery/Upload Date）；结果落 MySQL(状态)+ClickHouse(明细)，发 `data-ready` Kafka。资金盘/阵地数不可用时对应权重归零重归一化（AC-07）。

### forecast 的外部客户端

| Feign 客户端 | 外部系统 | 关键方法 | 用途 |
|-------------|---------|---------|------|
| `BigDataForecastClient` | 大数据团队 | `baseline(ForecastBaselineRequest)` | 实时预测基准（详见 AgentScope与预测文档 §预测基准契约） |

> 对象列表（区域/Dealer/门店/Promoter）与机型列表：一期复用 dataingest 已接入的 `TerritoryClient`/`ProductClient` 结果（经 dataingest facade 或直连外部，按数据归属定），**不新增重复 Feign**；归属在 forecast 模块设计期定。

### workbench 的外部客户端

| Feign 客户端 | 外部系统 | 关键方法 | 用途 | 遗留 |
|-------------|---------|---------|------|------|
| `DcrAppNotifyClient` | DCR APP | `notify(NotificationEventDTO)` | 发布/异常/复盘通知触发（异步，失败不阻塞、可重试） | **R4** 形态（REST/Kafka/SDK）；recipientId 映射 |
| `UacAuthClient` | UAC 权限 | `getUserGrants(userId)` | 四级权限授权数据（国家/品牌/角色/经销商） | 权限数据来源确认 |

`NotificationEventDTO`（稳定契约，跨 workbench/tracking 共用，置 common.dto）：
```
notificationType(TARGET_PUBLISHED/ALERT_RED/ALERT_YELLOW/REVIEW_GENERATED)
targetObjectId, targetObjectName, brand, countryCode, targetType
anomalyType?(执行告警), currentValue?, threshold?, triggerTime
periodStart?, periodEnd?, recipientIds[]（映射 DCR APP 用户 ID）
payloadExtra(Map，冗余字段)
```

### tracking 的外部客户端

| Feign 客户端 | 外部系统 | 关键方法 | 用途 | 遗留 |
|-------------|---------|---------|------|------|
| `DcrAppNotifyClient` | DCR APP | `notify(NotificationEventDTO)` | 红黄异常/复盘通知（共享 workbench 定义，避免重复） | R4 |
| `BigDataFeedbackClient` | 大数据团队 | `feedback(List<ModelFeedbackDTO>)` | 结果回流（异步，完整率≥95%，失败可重试） | **F2** 格式待对齐 |

`ModelFeedbackDTO`：targetPlanId, objectId, objectLevel, finalActualQty, finalAchievementRate, periodMonth, feedbackAt。

> `DcrAppNotifyClient` 定义置于 common 或独立 `notify` 子包，workbench/tracking 共用同一 Feign 接口（同一外部系统，符合"对外 Feign"边界，非模块间调用）。

## 三、REST 接口（前端 ↔ workbench/tracking，Controller 层）

> 契约基线只列资源清单，字段沿用上文 DTO；详细 OpenAPI 在各模块设计文档展开。统一经网关，`Accept-Language` 国际化，权限过滤在 Controller 入口。

| 前端页面 | 资源（示意） | 后端 |
|---------|-------------|------|
| 目标管理工作台 | `GET /goal/plans`（列表/筛选）、`GET /goal/workbench/summary`（全局指标/待办/风险/逾期） | workbench（全局指标转调 GlobalMetricsFacade） |
| 创建/拆解工作台 | `POST /goal/plans`、`GET /goal/plans/{id}/prediction`、`POST /goal/plans/{id}/confirm-total`、`GET /goal/plans/{id}/split`、`POST /goal/plans/{id}/split/{itemId}/resolve`、`POST /goal/plans/{id}/publish` | workbench（转调 Prediction/SplitFacade） |
| 流程配置 | `GET/PUT /goal/config/split-flow` | workbench |
| 执行追踪看板 | `GET /goal/tracking/board`、`GET /goal/tracking/gap`（下钻） | tracking |
| 复盘报告 | `GET /goal/reviews/{targetPlanId}` | tracking |
| 版本查询（二期 UI 预留） | `GET /goal/plans/{id}/versions` | workbench |
