# 技术设计：契约基线（二）接口清单

> 版本：v2.0（重定基线）| 日期：2026-07-23 | 作者：architect
> 定位：gate #1 产物。模块**门面接口**（应用内协作）+ **对外 Feign 接口** + **前端/APP REST 资源** + 核心 DTO 字段。
> 约束：门面接口入参/出参 DTO 均在 `common.dto`；Feign 仅用于外部系统；签名为契约，实现不得偏离。
> **v1.0 作废**：`PredictionFacade`、`SplitFacade` 的 rebalance/validate、`BigDataForecastClient`、`BigDataFeedbackClient`、`TerritoryClient`（阵地数）、`DrpClient.capitalPool`、复盘相关 REST 资源。

## 一、模块门面接口（`com.transsion.goal.<module>.facade`）

> Spring 同容器注入的 Java 接口，非 REST。返回值省略 Hulk 统一包装。异常经 `GoalBizException(GoalErrorCode)` 抛出。

### 1. dataingest —— `MetricQueryFacade`（本轮新增，替代 `HistoryDataQueryFacade` 的预测用途）

```java
public interface MetricQueryFacade {
    /** 批量取测算指标（供 calc 总目标测算与人员拆解）；缺失时抛 METRIC_UNAVAILABLE（附缺失清单） */
    Map<String, MetricValueDTO> getMetrics(MetricBatchQuery query);

    /** 数据可用性检查（测算前置 Step-A1），返回等级+跨度+缺失项，不抛异常 */
    DataAvailabilityDTO checkAvailability(AvailabilityQuery query);
}
```
`MetricBatchQuery`：countryCode, brandCode, periodMonth, `List<MetricKey>`, dimensionType(MODEL/PROMOTER/STORE/BRAND), dimensionIds?
`MetricKey` 取值（与需求 §4.4 一一对应）：`LAST_MONTH_SO` / `LAST_MONTH_DAILY_SO` / `LAST_MONTH_ACTIVATION` / `LAST_MONTH_DAILY_ACTIVATION` / `DAILY_ACTIVATION` / `CURRENT_STOCK` / `DOS` / `LAST_YEAR_SAME_PERIOD_SALES` / `SEASON_FACTOR` / `BP_AVG3M_SO` / `STORE_AVG3M_SALES`
`MetricValueDTO`：metricKey, dimensionType, dimensionId, value(BigDecimal), unit, **sourceSystem**, **fetchedAt**, availabilityLevel
> **`sourceSystem` + `fetchedAt` 是硬要求**，不是可选字段：需求 AC-01 要求测算界面展示每个因子的来源与取数时间，缺一个都过不了验收。

### 5. calc —— `EstimateFacade`（替代 `PredictionFacade`）

```java
public interface EstimateFacade {
    /** 触发总目标测算：匹配规则包 → 取指标 → 执行公式；同步返回（≤10s），失败抛 ESTIMATE_FAILED */
    EstimateResultDTO estimate(EstimateRequest request);

    /** 查询测算结果 */
    EstimateResultDTO getEstimate(Long targetPlanId);

    /** 机型维度分解：有货按公式分、缺货以可信库存为上限；Others 归集与重点机型强制单列 */
    ModelBreakdownDTO breakdown(ModelBreakdownRequest request);
}
```
`EstimateRequest`：targetPlanId, countryCode, brandCode, targetType, metricCode, targetPeriod
`EstimateResultDTO`：targetPlanId, formulaCode, **baseValue**, **calcDetail**（`List<CalcFactorDTO>`）, dataLevel(AvailabilityLevel), isDegraded, status(EstimateStatus), **adjustHints**（`List<AdjustHintDTO>` 微调因子提示）
`CalcFactorDTO`：factorName, factorKey, value, unit, sourceSystem, fetchedAt
`AdjustHintDTO`：hintType(PROMOTION/DOS/CLEARANCE_STOCK/CAPACITY/SEASON/PRICE_CHANGE), label, value?, description
`ModelBreakdownRequest`：targetPlanId, adjustedTotal, `List<ModelAllocation>`（modelCode, allocatedValue, isManual）
`ModelBreakdownDTO`：items[]（modelCode, modelName, isOthers, isKeyModel, isHsp, stockQty, credibleStockQty, allocatedValue, isManual）+ **coverageRatio**（分解到具体机型的占比，用于 AC-07 的 90% 校验）

> **已删除**：`conservativeValue` / `recommendValue` / `challengeValue` / `intervalLower` / `intervalUpper` / `confidence` / `basisText` / `predictionSource` 八个字段。出现即为 v1.0 残留。

### 6. calc —— `SplitFacade`

```java
public interface SplitFacade {
    /** 按规则包批量生成人员目标（SO总量 + HSP 双目标行）；无规则包抛 CALC_NO_RULE；
     *  全部 BP 历史销量为 0 抛 NO_HISTORY_SALES。异步执行，返回受理句柄 */
    SplitHandle split(SplitRequest request);

    /** 查询拆解结果（工作台列表页） */
    SplitResultDTO getSplitResult(Long targetPlanId);

    /** 批量导入覆盖；模板校验失败抛 IMPORT_VALIDATION_FAILED（附错误行清单），整批回滚 */
    SplitResultDTO importItems(ImportRequest request);
}
```
`SplitRequest`：targetPlanId, countryCode, brandCode, targetType, targetPeriod
`SplitResultDTO`：targetPlanId, status(SplitStatus), **coefficient**（本批系数，留痕便于复算）, items[]
每 item：promoterId, promoterName, storeId, storeName, **metricCode**, modelCode?, baseValue, generatedValue, finalValue, source(AUTO/IMPORT/MANUAL), **flags[]**（`MIN_FLOOR_APPLIED` / `STORE_TRANSFER_CALIBRATED` / `HISTORY_MISSING`）, version

> **已删除**：`rebalance(...)`、`validate(...)`、`ValidationResultDTO`、`BalanceMode`、`isLocked`、`level(ObjectLevel)` 层级字段（一期只有 PROMOTER 一层，不需要在 item 上带层级）。

### 7. calc —— `PrecheckFacade`（本轮新增）

```java
public interface PrecheckFacade {
    /** 人员目标预检：预计达成率区间 + 风险标记 R1/R2/R3；仅提示，不抛异常 */
    PrecheckResultDTO precheck(Long targetPlanId);

    /** 提示性校验：合计 vs 总目标×softCheckRatio；无同期总目标时返回 skipped=true */
    SoftCheckResultDTO softCheck(Long targetPlanId);
}
```
`PrecheckResultDTO`：targetPlanId, items[]（promoterId, metricCode, expectedRateLower, expectedRateUpper, riskFlags[]（R1_ABOVE_HISTORY / R2_FLOOR_APPLIED_LOW_HISTORY / R3_ABOVE_STORE_AVG））
`SoftCheckResultDTO`：**skipped**(bool，无同期总目标时 true), passed(bool), totalTargetValue, personSum, requiredValue, **gapValue**, gapRatio, thresholdRatio
> **契约级强调**：`softCheck` **永远不抛异常、永远不返回阻断标志**。它只是生成一份提示素材，是否继续由前端与用户决定。任何在实现中把它变成拦截点的写法都违反 E-7。
