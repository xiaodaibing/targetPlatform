# ADR-0002：数值预测由大数据团队提供（实时基准 + 本系统规则生成三档）

- 状态：已采纳
- 日期：2026-07-02
- 关联：`docs/requirements/待澄清问题清单.md`（决策 F2、OQ-04、OQ-DI-01）、`docs/requirements/AI预测与拆分.md`

## 背景

平台需要为目标制定提供数值预测（三档目标建议 + 推荐区间 + 置信度）。PDF 原稿设想由本系统用 Prophet/SARIMA/KNN/衰减曲线自建预测模型，这需要数据科学家、特征工程、模型训练与调优，对小团队负担重；且公司已有大数据团队具备建模能力与自有数据湖。

## 选项

1. **本系统自建预测模型（Python 生态，作为 AgentScope 工具或独立算法服务）** — 优点：预测逻辑自控。缺点：需数据科学家/特征工程/训练调优；跨语言（Python）落地复杂；与大数据团队能力重复。
2. **预测完全由大数据团队提供（含三档）** — 优点：本系统零建模。缺点：三档口径（保守/挑战幅度）是业务策略，交由外部团队不灵活。
3. **大数据团队提供基准值，本系统按配置规则生成三档/区间** — 优点：建模外部化（本侧不需数据科学家），同时三档幅度作为业务规则由本系统灵活配置；职责清晰。缺点：新增对大数据团队的实时接口依赖。

## 决策

选 **选项 3**：

- **大数据团队实时调用返回预测基准值**；本系统**按配置规则在基准上生成三档（保守/推荐/挑战）与推荐区间**（三档生成规则引擎是本系统构建的能力）。
- 本系统**不自建预测模型、不做特征工程/模型训练、不需本侧数据科学家**；特征工程与建模由大数据团队负责。
- **大数据团队使用其自有数据湖，本系统无需向其开放数据出口**（OQ-DI-01 闭环）。
- 执行结果回流给大数据团队用于其模型迭代（接口/格式设计期对齐）。

## 后果

**正面**：本侧无建模负担，人力聚焦业务闭环；三档策略可配置；数据接入模块无需承担对大数据团队的数据出口。

**负面 / 代价**：
- **实时预测接口成为关键依赖**：延迟/可用性直接影响目标创建流程——须与大数据团队定接口契约 + SLA；预测不可用时**降级为人工设定**，不阻塞流程。
- 置信度来源需明确（随基准返回 or 本系统规则映射）。

**后续待办（设计期 / 接口契约阶段）**：
- 与大数据团队定**实时预测基准接口**：入参（国家/品牌/周期/对象/机型）、出参（基准值、置信度、口径）、SLA（延迟/可用性/限流）。
- 定"三档 + 推荐区间"的配置规则模型（幅度、按国家/品牌/目标类型可配）。
- 定结果回流（ModelFeedback）给大数据团队的接口与格式。
- 无预测/超时的降级交互（提示 + 人工设定）落到设计。
