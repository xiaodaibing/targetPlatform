# ADR-0004：LLM 数据出境与合规方案

- 状态：**已废弃（2026-07-23，第四轮澄清 E-3）**
- 原状态：提议（待安全 / 法务评估）
- 日期：2026-07-02　废弃日期：2026-07-23
- 关联：ADR-0001（已废弃）、`docs/requirements/第四轮澄清-MVP重定基线.md`（E-3）

## 废弃说明

本 ADR 的存在前提是 ADR-0001「一期引入 LLM」。第四轮澄清确认一期不引入任何 LLM / Agent 能力，因此**一期不存在向 LLM 提供商发送渠道 / 门店 / Promoter / 销售数据的场景**，数据出境风险随之消失。原 D-5 决策新增的「市场情报网搜工具」出站通道亦一并作废。

**注意：本 ADR 废弃的只是「LLM 出境」这一条通道，不等于平台没有数据合规义务。** 一期仍需遵守的合规要求（不因本 ADR 废弃而免除）：

- 孟加拉渠道与人员数据的存储、访问与权限隔离（人员级隔离：BP 只能查看本人数据）
- 与 DCR APP、公司 BPM 等外部系统交互时的数据最小化原则
- 操作日志与审计留痕的保存期限要求

以上应在架构设计的安全章节中承接，不依赖本 ADR。

## 背景（历史记录，仅供二期参考）

原判断：LLM 需接收上下文数据以生成业务依据 / 复盘 / 策略。一期试点孟加拉，涉及渠道 / 门店 / Promoter / 销售数据，可能含 PII。发往 LLM 提供商（如 DashScope，服务节点可能跨境）存在数据出境与隐私合规风险。

## 选项（历史记录）

1. 公有云 LLM + 发送前脱敏 / 最小化 — 接入快，但数据出境风险，聚合数据仍可能敏感
2. 区域化 / 私有化模型部署 — 合规风险低，部署运维成本高、模型能力可能弱
3. 混合：非敏感文本走公有云，含 PII 走本地脱敏或私有化模型 — 平衡，但需数据分级与路由

原建议方向为选项 3，最终选型交安全 / 法务 + 大数据团队评估。

## 后果（废弃后）

**正面**：移除一期的跨境合规评估与脱敏中间件工作量；不再需要安全 / 法务的前置评估作为上线卡点。

**后续待办**：
- [ ] 二期若重启 LLM 能力，本 ADR 需重新开启评估，且必须在功能开发前完成，不得事后补
- [ ] 架构设计的安全章节需独立承接「人员级数据隔离」「外部系统交互数据最小化」「日志留存期」三项要求
