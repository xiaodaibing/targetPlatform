需求规格路径：`docs/requirements/目标协同管理.md`（2026-07-02 产出，v1.0）

**Why:** 记录本模块核心决策与非显然约束，供后续 architect/implementer 快速定位关键细节。

**How to apply:** 进入技术设计或实现前，读本记录 + 需求规格文件，不要仅凭 PDF 设计。

## 关键决策

- "是否审批" UI 固定显示为"否"且禁用，附提示"二期上线"——不是删掉字段，是预留入口
- "是否关联返利" 字段一期完全不展示（C6：返利不涉及）
- "目标对象" 一期仅 Dealer / 门店 / Promoter，不含 Supervisor（R6）
- 版本留痕仅后端存储，预留版本查询 API 供二期接入
- 通知触发异步解耦，失败不阻塞发布
- 强校验逻辑属 AI 预测与拆分模块，本模块在发布前调用其接口触发校验

## 核心实体（供 architect 建表参考）

- 目标（Target）：基础信息 + 状态机
- 拆分结果值（TargetValue）：各层级/对象目标值 + 确认状态
- 流程配置（ProcessConfig）：三维度（国家/品牌/目标类型）可配置项
- 版本快照（TargetVersion）：6 种版本类型
- 操作日志（OperationLog）：所有关键动作审计
- 通知触发记录（NotificationRecord）：触发状态 + 失败原因

## 目标状态机（本模块视角）

草稿 → 总目标确认中 → 拆分中（AI 拆分模块控制）→ 拆分结果确认中 → 已发布

## 跨模块依赖

| 依赖 | 类型 | 说明 |
|------|------|------|
| AI 预测与拆分模块 | 阻塞 | 预测接口 + 拆分接口 + 强校验接口 |
| 执行追踪模块 | 非阻塞 | 工作台全局指标/风险预警数据来源 |
| 权限管理支撑系统 | 阻塞 | 四级权限数据 API |
| DCR APP 通知接口 | 非阻塞（异步） | R4 设计期依赖，形态待定 |
