## 一期（MVP）确认基线（2026-07-02 两轮澄清已闭环）

**主流程**：
```
数据接入(DRP/DCR/激活/库存)
→ AI 预测三档总目标
→ PC 端人工确认总目标
→ 规则拆分(仅简单规则/无市场因子)
→ PC 端确认拆分(过强校验)
→ 直接发布生效（无审批）
→ 触发通知告知对象(DCR APP，仅触发)
→ 执行追踪(Gap/红黄绿)
→ 基础复盘
```

**明确移出一期**：
- 审批流 / 加签 / 转交（→二期，接 BPM）
- 对象移动端确认与申请调整（→二期，DCR APP RN 团队负责）
- 超时 Pass / 催办（→二期）
- 返利（完全不涉及）
- 防作弊 / 串货检测（允许跨区域，不做）
- 版本对比 UI（仅后端留痕）
- AI Agent / AI 对话助手（二期）
- 市场容量 / 机型生命周期拆分因子（二期）
- "超市场容量"异常识别（市场库二期）
- "新品供应风险"异常识别（工厂产能数据二期）

**指标简化**：
- 北极星 & 合理性统一 = "目标达成率占比"
- 不做：预测准确率指标、返利偏差率

**移动端**：一期仅触发通知，由既有 DCR APP（React Native，专门团队）承接展示；本平台不建移动端。

**Why**：产品经理两轮确认（R1-R6），PDF 原稿 MVP 范围被显著收敛。
**How to apply**：所有模块需求规格的一期范围判断，以本文件为准，不以 PDF 为准。
