AI目标管理平台一期（孟加拉试点）项目基线，两轮澄清已于2026-07-02闭环。

**Why:** 需求规格阶段的权威口径来源是`docs/requirements/待澄清问题清单.md`的"沟通结论回填区"，与PDF冲突时以该文件为准。

**How to apply:** 写任何模块需求前先读该文件，不要从PDF提取结论。

## 一期主流程（R1已确认）
数据接入(DRP/DCR/激活/库存) → AI预测三档总目标 → PC端人工确认总目标
→ 规则拆分(仅简单规则/无市场因子) → PC端确认拆分(强校验) → 直接发布生效
→ 触发通知告知对象(DCR APP，仅触发) → 执行追踪(Gap/红黄绿) → 基础复盘

## 关键术语
- 客户 = Dealer（经销商）
- 人员（一期）= Promoter（R6确认，Supervisor/区域经理预置但非一期主要对象）
- 目标类型：SI（进货）/ SO（销出）/ ST（零售）
- 拆解层级：国家→区域→目标对象(Dealer/门店/Promoter)→机型
- 品牌：TECNO / Infinix（itel 二期）
- 一期试点国家：孟加拉

## 明确移出一期的内容
- 审批流/加签/转交（→二期BPM）
- 对象移动端确认与申请调整（一期仅告知）
- 超时Pass/催办机制
- 返利（完全不涉及）
- 防作弊/串货检测
- 版本对比UI（仅后端留痕）
- AI Agent/对话助手
- 市场容量/机型生命周期因子拆分
- 预测准确率北极星指标（仅用"目标达成率占比"）

## 遗留至设计阶段的依赖
- R2：SI/SO/ST精确取数口径（DCR接口契约阶段钉死，ST vs SO差异待确认）
- R3：审批对接BPM（二期，架构预留）
- R4：通知触发方案（与DCR APP团队对齐）

## 风险登记
- R5（数据体检缺位）：孟加拉12个月历史数据存在但无体检结论，AI预测质量存在不确定性。建议M0内数据团队出体检报告作为预测建模前置门槛。
