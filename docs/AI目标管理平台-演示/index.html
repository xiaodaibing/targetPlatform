§0 · 一页纸速览

一屏说清：痛点、解法、最快·最能·最低成本

痛点

渠道目标制定靠人工拍脑袋（2–3 天/次），拆解到 Dealer/门店/Promoter 层层加码易失真；执行追踪滞后、异常发现晚；复盘全凭经验。跨 TECNO/itel/Infinix 多国渠道，规模一大就失控。

解法

单体应用 + 包级模块闭环：数据接入与达成计算 → AI 三档预测与规则拆分 → PC 双工作台确认发布 → 执行追踪与复盘回流。AI 只做建议、全程人工确认（HITL）。

最快

单体 + 包级模块免分布式税；复用公司 Hulk/Vben/Eureka/AgentScope 现成基座；预测外包，不建数据科学栈；契约先行使模块并行。

最能

全流程闭环 + 102 条可判真假的验收标准；AI 辅助但不越权；幂等/降级/权限/审计在设计层已成体系。

最低成本

不自建预测模型（免数据科学家）；LLM 混合合规按需；单体省运维。

可信度

已完成：PRD 评审闭环 → 102 条 AC → 4 ADR → 7 份设计文档（到建表级）→ 主 agent 跨模块一致性复核（修正 4 处真实问题）。公司级脚手架（Hulk 后端 26 个基座类 + Vben 前端）就绪。

主轴：最快 · 最能 · 最低成本——把渠道信任当一等公民，AI 克制而不越权。

§3 · 智能 / 算法单元设计

一期「智能」由三类单元构成——不把规则引擎包装成 Agent

如实说明：只有第 3 类是 LLM Agent。数值与文本解耦，LLM 失败时数值链路照常，文本降级为规则模板。

#单元类型一期职责是否 LLM
1大数据预测基准外部统计/建模（大数据团队）实时返回预测基准值（特征工程/建模在其侧）否（外部）
2三档规则引擎 TierRuleEngine确定性规则在基准上按配置生成保守/推荐/挑战 + 区间 + 置信度映射否
2拆分/平衡/强校验/异常引擎确定性规则/数值逐层拆分、四模式平衡、CV-1/2/3 校验、A1–A4 异常否
3forecastBasisAgentLLM Agent（AgentScope）生成「业务语言依据」文本，HITL是
3reviewAgentLLM Agent（AgentScope）生成复盘「优化建议」文本，HITL是

工具与知识调用

工具 history_stat / review_stat（@Component，经门面接口只读取数，不直连别模块表）；每次 call 必传 RuntimeContext(userId, sessionId)。

出境合规

所有发往 LLM 的数据必经 PiiRedactionMiddleware（唯一出口，PII→稳定 ID）+ LlmAuditMiddleware 审计；Provider 受 ADR-0004 约束。

红线：所有数值（三档/拆分/校验/达成/异常）均为规则计算，不经 LLM；LLM 失败时数值链路照常，文本降级规则模板。AI 只建议、不决策。

§4 · Spec 设计示例（最具差异化的核心功能）

AI 三档预测 + 人工确认——有据可依地拍板，不被虚高置信度误导

User Story

作为国家/品牌经理，我要在创建目标时看到 AI 三档预测（保守/推荐/挑战）+ 推荐区间 + 置信度 + 业务依据，以便有据可依地拍板总目标；当数据不足时得到明确降级提示，不被虚高置信度误导。

门面 PredictionFacade.predict(req) → 异步受理
getPrediction(planId) → 前端轮询，status 驱动 UI
REST: GET /goal/plans/{id}/prediction
错误码: PREDICTION_FAILED（透传+降级手输）

数据模型 t_target_prediction（追加式留痕）：baseline_value, conservative/recommend/challenge_value, interval_lower/upper, confidence, basis_text, prediction_source, status, is_current（历史行即 AC-22 快照）。

业务规则 / 验收标准（可判真假）

  • BR-1（AC-05）：六项输出全有或全空；违反单调性（保守≤推荐≤挑战、下限≤推荐≤上限）→ 整体 FAILED 降级人工
  • BR-2（AC-02）：数据 LIMITED 时展示「数据不足，仅供参考」，置信度不得超过 LIMITED 上限（禁伪精确）
  • BR-3（AC-04）：大数据超时/无返回 → DEGRADED，允许跳过 AI 人工输入总目标，跳过动作留痕
  • BR-4（AC-20）：预测区展示「仅供参考，最终目标由您决定」，无「AI 已自动确认」表述

异常场景：大数据超时→DEGRADED 降级；三档单调性违反→FAILED 留空降级人工；LLM 依据失败→basisText 回退规则模板，数值链路仍 COMPLETED。

BR-1~4 对应 forecast AC-01/02/04/05/20/22，均为可判真假陈述，直接转测试用例（JUnit5 + Mockito 对门面/Feign/Agent 打桩）。

明确不在 MVP

不自建预测模型、不自建移动端、不做 AI 自动确认/自动发布、不做逾期催办闭环/超时 Pass 考核/严格相等拆分等二期增强。一期先把闭环、可信、可上线做透。

如何推进到上线

设计已完成、契约已锁、脚手架就绪；外部真实接口未通期间用 Feign Mock / 契约桩先跑通闭环。契约先行使各模块里程碑可并行/交叠推进，而非严格串行。
